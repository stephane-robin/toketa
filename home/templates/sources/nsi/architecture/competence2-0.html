Nous allons explorer le Terminal et apprendre à l'utiliser le bash en suivant des directives étape par étape.

Toutes les instructions nécessaires à ce travail pratique se trouve dans le chapitre Le système Linux.

Rappel, pour naviguer dans l'éditeur de texte nano, il faut comprendre Ctrl+O pour ^O et penser à confirmer les demandes avec la touche Entrée.

Activité 1

Astuce : n'oubliez pas que la touche de tabulation utilisée dans le terminal permet l'autocomplétion des instructions ou encore des noms de fichiers connus.

  • Ouvrir le Terminal puis se rendre dans le dossier Documents,

  • Afficher tous les fichiers et sous-dossiers présents dans Documents, avec les détails et les fichiers cachés.

  • Creer un sous dossier appelé Dossier_term

  • Se rendre dans Dossier_term, puis y créer un fichier vide appelé fichier1.txt

  • Ouvrir fichier1.txt avec nano, puis écrire dans ce fichier le texte "Les sanglots longs de l'automne blessent mon coeur d'une langueur monotomne."

  • Quitter nano, puis lire le contenu de fichier1.txt dans le Terminal

  • Copier fichier1.txt puis le coller dans Documents sous le nom fichier2.txt

  • Ouvrir fichier2.txt avec nano, puis en modifier le contenu pour que soit écrit print("Les sanglots longs de l'automne ...")

  • Renommer fichier2.txt en fichier3.py en respectant bien le changement d'extension.

Activité 2

Nous allons apprendre à utiliser simplement la boucle for en bash.

  • Ouvrir le Terminal puis se rendre dans le dossier Documents.

  • Pour comprendre l'utilisation de la boucle for, recopier puis exécuter le code suivant :

    for i in 0 1 2 3
do
echo "Hello $i"
done

  • En suivant l'exemple du code ci-dessous créer 5 fichiers appelés fichier1.txt, fichier2.txt, fichier3.txt, fichier4.txt et fichier5.txt.

  • Modifier le nom de ces fichiers en utilisant toujours une boucle for, de sorte que fichier1.txt devienne fichier1-0.txt.

Savoir modifier les noms d'un ensemble de fichiers à partir du bash est très utile et peut permettre de gagner énormément de temps lorsqu'on change la structure d'un grand projet.

Activité 3

Nous allons apprendre à exécuter un fichier Python à partir du Terminal. Cela peut être utile lorsqu'un programme s'exécute mal ou ne s'exécute pas avec notre environnement de développement (comme VS Code par exemple) alors qu'il devrait s'exécuter et qu'aucune erreur de code n'est détectée.

Astuce : n'oubliez pas que les flèches de navigation Haut, Bas permettent de retrouver les dernières commandes exécutées sur le bash afin d'éviter de devoir les retaper.

  • Ouvrir le Terminal et se rendre dans le dossier Documents

  • Créer un fichier appelé essai.py

  • Ouvrir ce fichier à l'aide de nano puis taper le code suivant

    for _ in range(3):
    print("il connaît pas Raoul ce mec")

  • Quitter nano et rester dans le dossier Documents

  • Exécutez votre fichier Python en tapant python3 essai.py. On peut donc facilement exécuter tout type de fichier Python dont on connaît l'emplacement dans l'ordinateur.

  • Retourner à la racine du Terminal puis taper simplement python3 pour accéder à une console Python

  • Taper maintenant print("Pas de chichi entre nous, appelez-moi Super") puis exécutez avec la touche Entrée ou encore 3 + 2 puis exécutez avec la touche Entrée. Voilà, vous êtes capable d'exécuter une console Python sur votre Terminal.

  • Quittez maintenant la console Python pour revenir au bash du Terminal

  • Python n'est pas le seul langage connu de votre ordinateur. Bien souvent Java est déjà présent dans votre machine. Si c'est le cas, il suffit d'exécuter l'instruction java --version pour en obtenir la confirmation. Si java est présent sur votre ordinateur, vous pouvez ouvrir le fichier essai.py avec nano puis en modifier le contenu de la façon suivante :

    class Test{
public static void main(String[] args)
{
System.out.println("c'est cela oui")
}
}

    Vous venez d'écrire votre premier code java qui se contente d'afficher une phrase.

  • Quittez ensuite le fichier essai.py en enregistrant les modifications effectuées.

  • Renommer le fichier essai.py en essai.java pour que l'ordinateur comprenne que votre fichier est écrit en java.

  • Il ne reste plus qu'à exécuter l'instruction java essai.java au Terminal pour exécuter du java. On a donc simplement remplacé python3 par java. Ceci est également valable pour de nombreux autres langages de programmation.
