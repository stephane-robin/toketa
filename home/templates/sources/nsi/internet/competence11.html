Introduction

A vous de jouer : la plupart des risques encourus sur Internet sont liés aux utilisateurs. expliquez dans le détail comment il est possible de se faire piéger en se rendant sur un site, en répondant à un email, en consultant son smartphone, ... Quels sont donc les comportements à risque à éviter ?

La cybercriminalité peut prendre différentes formes

Les principales menaces en cybercriminalité sont les suivantes :

  • vol d’information, et parfois usurpation d’identité
  • destruction des données ou des ordinateurs, smartphones
  • défiguration de site Web (defacement)
  • modification d’information par un utilisateur non autorisé
  • déni de service DoS
  • publicité indésirable
  • escroquerie

Le mot malware est un terme général qui fait référence à un logiciel malveillant.

Le phishing

Il consiste à se faire passer pour une personne, un organisme ou une entreprise de confiance afin de demander des informations confidentielles à un victime. Celle-ci est amenée à cliquer sur un lien envoyé par email lui donnant accès à un site web ressemblant au site officiel, faisant ainsi croire à l’utilisateur qu’il navigue sur un site de confiance. Lorsque l'utilisateur tape ses informations confidentielles, celles-ci sont dérobées à son insu.

Précautions à prendre :

  • vérifier l'adresse d'origine de l'email, il est facile de confondre tresor-public@administration.fr et tresor-public@adrninistration.fr
  • ne pas cliquer sur des liens dont la provenance n'est pas certifiée,
  • le cas échéant vérifier l'adresse du site sur lequel le lien nous a renvoyé,
  • vérifier l'orthogaphe dans le texte de l'email, il est fréquent que les emails frauduleux comportent de nombreuses fautes d'orthographes.

Les virus

Il s'agit de logiciels malveillants installés sur l'ordinateur ou le smartphone d'une victime, le serveur d'un site Web ou d'une entreprise pour voler des informations, détruire le fonctionnement d'une machine ou d'un serveur et fournir un prétexte pour une rançon.

Les différentes contaminations possibles sont les suivantes :

  • par les supports amovibles (clés USB)
  • par les pièces jointes des emails
  • par les fichiers exécutables (extensions .EXE sur Windows, .PY s'il s'agit d'un script Python)
  • par les fichiers multimedia .MP3, .MP4, .AVI, …

Précautions à prendre :

  • ne pas télécharger des applications dont on ne connaît pas la réputation,
  • télécharger des applications à partir de la page officielle de la société qui la développe,
  • installer un antivirus sur son smartphone, sa tablette et son ordinateur,
  • "les Mac et les Iphones sont protégés naturellement des virus" est une légende urbaine,
  • scanner les supports amovibles à l'aide d'un anti-virus avant de les utiliser.

Les worms

Il s'agit d'un type d'attaque particulier qui utilise un carnet d'adresse pour se propager. Lorsque le hacker a obtenu l'accès au carnet d'adresse d'un compte WhatsApp, Google, Outlook, ..., il infecte les autres membres du réseau, sans se dévoiler ou en se faisant passer pour un ami au sein du réseau.

Les trojans

Il s'agit d'un certain type de virus qui s'insèrent dans une machine puis donnent accès aux ressources de cette machine à l'insu de son propriétaire.

Les spywares

Il s'agit d'un terme général relatif à un malware qui espionne les actions d'un utilisateur sur son ordinateur ou son smartphone, pour récupèrer des informations à partir d'une machine infectée. Ce vol d'information peut ensuite être utilisé pour usurper l'identité de quelqu'un afin de créer des abonnements ou de faire des achats au nom d'un autre, d'ouvrir des prêts bancaires. On parle dans ce cas de spoofing.

Les botnets

Il s'agit d'un ensemble de robots informatiques qui se déploient sur de grandes quantités de machines pour les utiliser comme relais d'attaques ou de spams. On parle également de réseaux zombies.

Le déni de service

Il consiste à paralyser un réseau, un serveur ou un site Web pour que les utilisateurs ne puissent plus en obtenir l'accès. D'innombrables requêtes sont transmises en même temps à un serveur pour le saturer. Le déni de service distribué utilise plusieurs machines relais à partir d'une seule machine pour saturer un système.

Il arrive que parfois le déni de service n'entraîne qu'un ralentissement du système. On parle dans ce cas de downtime.

Le déni de service s'accompagne généralement d'une demande de rançon.

La publicité malveillante

Les Spams

Il s'agit d'une technique de marketing consistant à diffuser massivement par email de la publicité non sollicitée par les utilisateurs.

Les adware

Il s'agit d'un logiciel qui présente des publicités non sollicités, souvent sous forme de fenêtre pop-up.

Précautions à prendre :

  • utiliser un ad-blocker

Les browser-hijackers

Il s’agit de modifications du navigateur sans la permission de l’utilisateur pour afficher des publicités ou enregistrer l’activité de l’utilisateur. Un browser-hijacker peut par exemple être utilisé pour générer de fausses alertes de sécurité afin d'inciter à acheter un anti-virus.

Le Scam

Il s'agit d'une forme d'escroquerie qui utilise les technologies modernes pour exploiter la naïveté des utilisateurs afin qu'ils fournissent de leur plein gré des données confidentielles ou de l'argent. Le hacker peut se faire passer pour un agent de l'état, un membre de l'organisation ou l'entreprise pour laquelle il travail, un membre de sa famille, une personne en détresse, ...

Exploitation de vulnérabilités

L’exploitation de vulnérabilité consiste à abuser de failles connues dans les navigateurs, les applications web, les réseaux, les systèmes d'exploitation et les applications de la machine.

L'une des techniques d'exploitation de vulnérabilité les plus utilisées sur le Web est l'injection SQL, qui consiste à insérer dans un formulaire non protégé du code SQL capable d'altérer ou de détruire la base de données sur laquelle évolue le site Web attaqué.

Une autre technique de hacking sur le Web très utilisée est le Cross Site Scripting XSS, qui injecte du code côté client sur les pages Web visitées par d'autres utilisateurs, et permet de se connecter en tant qu'un autre utilisateur sans connaître le mot de passe de cet utilisateur.

Toutes les vulnérabilités recensées sont enregistrées dans le répertoire du CVE afin d'en conserver l'historique et le mode opératoire. Tout individu peut contribuer à cette base de données en publiant une nouvelle vulnérabilité.
