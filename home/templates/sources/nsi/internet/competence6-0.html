Les langages du Web

Pour construire un site Web, on utilise plusieurs types de langages : des langages de description tels que HTML et CSS qui sont au coeur du fonctionnement du Web et définissent la structure de chaque page visitée, des langages de programmation tels que JavaScript, PHP, Java, Python,... et un langage de traitement des bases de données tels que SQL ou MongoDB.

Pour mieux comprendre ce que fait chaque langage, on peut simplifier les explications de la façon suivante :

  • HTML gère le contenu des pages

  • CSS gère le style des pages

  • PHP, Python, Java... gère les autorisations d'accès, les affichages conditionnels, les différents traitements demandés, et l'accès aux bases de données

  • JavaScript gère la convivialité et l'interactivité du site avec l'utilisateur

  • SQL gère le stockage des données et leur récupération

Chacun de ces langages va interagir avec un interlocuteur différent. Certains vont être interprétés par un client (le navigateur), d'autres par un serveur. Ils ont tous leur importance et sont complémentaires. C'est un peu comme une équipe parlant plusieurs langues mais capable de travaille ensemble pour arriver à un objectif commun.

  • les langages interprétés par un client (on parle de langages frontend) tels que HTML, CSS et JavaScript décrivent comment le site Web doit s'afficher. Ils peuvent par exemple afficher un menu clignotant dans une page Web, proposer un quiz ou afficher une image en fonction des réponses fournies par l'utilisateur. Ces languages ont un impact visuel important et rendent le site Web convivial.

  • les langages interprétés par un serveur (on parle de langages backend) tels que PHP, Java, Python, JavaScript ... décrivent comment le site Web doit se comporter en appliquant des régles définies par le programmeur. Ils peuvent par exemple n'afficher la page Web que si l'utilisateur s'est identifié ou enregistrer dans une base de données un résultat fourni par l'utilisateur. Ils réalisent donc l'interface avec le langage de requête de bases de données

  • les langages de gestion de base de données tels que SQL et MongoDB s'occupent exclusivement de stocker et gérer les bases de données du site Web.

Attention, JavaScript est une particularité car il peut interagir tout aussi bien côté client que côté serveur. Il s'impose donc petit à petit comme LE langage privilégié du Web.

En résumé

Les échanges de données sur Internet utilisent des protocoles. Le Web, quant à lui, utilise ces protocoles mais fonctionne en respectant une structure client-serveur.

Part de marché des différents langages sur le Web

Plutôt que de commencer avec une page blanche pour construire un site Web, on préfère utiliser des CMS Content Management System, qui sont des modèles prêts à être modifiés. Les CMS les plus utilisés en terme de part de marché en 2023 sont (source Kinsta) :

  • WordPress 40% de tous les sites Web (open-source, offre de nombreux thèmes et plug-in, modifiable en PHP)

  • Joomla 3% de tous les sites Web (open-source, offre plus de fonctionnalités que WordPress, adapté aux site professionnels, modifiable en PHP)

  • Drupal 2% (destiné essentiellement aux professionnels, modifiable en PHP)

  • Magento (spécialisé dans les gros sites de e-commerce)

  • Typo3 (sépare la partie front-end de l'interface administrateur)

  • Wix (idéal pour débuter)

On peut également utiliser un frameworks, qui est une boîte à outils pratique, munie de multiples fonctionnalités pour faciliter le travail de développement. Les frameworks permettent en général de construire un site en respectant une structure qui facilite le partage des tâches, comme par exemple le design pattern MVC de Laravel ou le design pattern MVT de Django. Les principaux frameworks pour le Web sont les suivants :

  • Laravel, Symfony et Zend pour PHP

  • NodeJS et Angular pour JavaScript

  • Java EE pour Java

  • Django pour Python

  • ASP.NET pour C#

  • Ruby on Rails pour Ruby

En 2023, PHP est le langage de programmation le plus utilisé sur le Web avec 79% des parts de marché. Ceci s'explique car les principaux CMS et les principaux frameworks sont écrits en PHP. Toutefois, il est souvent critiqué et sa côte de popularité chute lentement pour privilégier des langages comme JavaScript ou Python.

Architecture trois tiers

Les bases de données

Les bases de données permettent d'enregistrer des informations afin de les ré-utiliser lors d'une prochaine session sur un site. Elles fonctionnent avec un autre serveur appelé serveur de base de données. L'accès à ce serveur s'effectue à l'aide d'un langage de programmation côté serveur tel que Python, Java, PHP, ...

En revanche, la gestion des données du serveur (choisir un emplacement de stockage, nommer les tables, savoir récupérer des éléments) s'effectue à l'aide d'un Système Gestionnaire de Bases de Données SGBD, qui est un logiciel communiquant principalement en langage SQL. Ainsi, un langage tel que Python peut intégrer du code SQL dans ses requêtes pour servir d'interface entre le programme principal et le gestionnaire de bases de données.

Les principaux systèmes de gestion de base de données SQL sont les suivants :

  • MySQL

  • PostgreSQL

  • SQL Serveur

  • Oracle

  • SQLite

Les bases de données de type SQL sont dites relationnelles. Leur structure est complexe et autorise difficilement la scalabilité. Avec l'avènement du Big Data et des bases de données orientées en graphe, sont nés de nouveaux types de bases de données, dites non relationnelles, qui n'utilisent plus le langage SQL mais un langage propre à chaque système. Par exemple MongoBD ou Cassandra créé par Facebook.

Une architecture 3 tiers pour générer une page Web

Nous avons appris qu'Internet était construit suivant un modèle en couches, et que le fonctionnement du Web était basé sur une structure client-serveur. Nous venons de voir qu'il existe également un autre serveur pour les bases de données, ce qui donne en fait au Web une architecture 3 tiers.

L'interaction entre un client et un serveur est la suivante :

  • le client demande une page au serveur du site

  • le serveur du site effectue une requête auprès du serveur de base de données pour obtenir certains résultats qui ont été conservés en mémoire

  • le serveur du site génère finalement la page en fonction des conditions requises par l'application utilisée, en transformant le code source (par exemple PHP) en HTML, puis il envoie la page créée au client

  • le client lit la page à l'aide du navigateur et utilise éventuellement JavaScript si une interaction dynamique est nécessaire côté client

En résumé

La structure client-serveur cache en son sein un autre serveur appelé serveur de base de données. Ainsi, toute requête sur le Web utilise une architecture 3 tiers.
