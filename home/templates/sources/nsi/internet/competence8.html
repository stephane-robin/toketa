Prérequis : de bonnes connaissances en HTML

Structure du code

JavaScript est un langage interprété qui se veut simple et dont le but initial était de rendre dynamiques et interactives les pages Web.

Les commentaires sont indiqués à l'aide de // pour les commentaires simples et /* */ pour les commentaires sur plusieurs lignes.

Comme beaucoup de langages, JavaScript utilise un ; à la fin de chaque instruction.

Dans une page HTML, il est préférable d'appeler un fichier JavaScript à la fin d'un document (avant la balise </body>) pour éviter de ralentir le chargement de la page.

<!DOCTYPE html>
<html>
<head>
  <meta charset="utf-8">
  <title>Ma page web</title>
</head>

<body>

<!-- corps du document -->

<script src="fichierJavaScript.js"></script>

</body>
</html>

Hello world

Commençons par le traditionnel "Hello world". Créer un fichier minimaliste testJS.html comme ci-dessous : 

<!DOCTYPE html>
<html>
<head>
  <meta charset="utf-8">
  <title>Ma page web</title>
</head>

<body>

<script src="testJS.js"></script>

</body>
</html>

Dans un fichier testJS.js placé dans le même dossier que testJS.html, écrire le code suivant :

alert("Hello world");

L'exécution du code se fait directement lors de la lecture du fichier page.html par le navigateur Web Chrome, Edge, Opera, Safari, ...

Ici, le message apparaît à l'intérieur d'une fenêtre pop-up.

Il est possible de visualiser le résultat dans la console du navigateur. Pour cela, il faut ouvrir dans le navigateur l'onglet Outil de Développement, puis l'onglet Console. Il faut également modifier le script dans le fichier testJS.js en écrivant cette fois :

console.log("Hello world");

Les variables

Types

JavaScrpt est un langage à typage dynamique, i.e. qu'il identifie lui-même les types de variables et autorise les changements de type. La déclaration des variables n'est donc pas nécessaire.

Par ailleurs, JavaScript est un langage faiblement typé, car il convertit automatiquement les types de variables. Ainsi, la concaténation d'un String avec un nombre transforme implicitement la variable obtenue en String. Pour éviter tout changement intempestif pouvant conduire à une erreur, il faut donc systématiquement tester le type des variables avant de les utiliser.

Les principaux types de variables sont : number pour les nombres (entiers ou réels, positifs ou négatifs), string pour les chaînes de caractères, boolean composés de true et false (sans majuscule), Null et undefined.

Pour tester le type d'une variable, on utilise l'instruction

typeof maVariable

Si cette instruction renvoie undefined, c'est que la variable n'existe pas (elle n'a pas été déclarée).

Il est tout à fait possible de caster une variable. L'instruction parseInt(chaine) permet de transformer un String en nombre si cela est possible alors que l'instruction toString(nombre) permet de transformer un nombre en String.

L'instuction Number(x) permet également de transformer un String en Number lorsque cela est possible, et inversement l'instruction String(x) permet de transformer un nombre en chaîne de caractères.

Affectation d'une valeur

Pour affecter une valeur à une variable, on utilise préférentiellement le mot-clé let plutôt que var qui désigne une variable locale, et s'utilisait essentiellement avec les anciennes versions de JavaScript.

let maVarible = 12;

Notons que JavaScript est sensible à la casse des caractères.

Les constantes

On définit une constante avec le mot-clé const au lieu de let.

La portée des variables

Les variables définies avec le mot-clé let ont une portée (un scope) sur le bloc dans lequel elles ont été définies, et non pas une portée globale. Par exemple, le code suivant génère une erreur car la variable n'est pas définie que dans la condition if et non pas pour le programme principal.

let estConnecte = true;

if (estConnecte){
  let message = "bienvenue";
}
else{
  let message = "vous n'êtes pas connecté"
}
console.log(message); # renvoie une erreur ici

Pour reprendre l'exemple précédent, il faut donc déclarer la variable message de façon globale comme dans le code ci-dessous :

let estConnecte = true;
let message = "";

if (estConnecte){
  message = "bienvenue";
}
else{
  message = "vous n'êtes pas connecté"
}
console.log(message); # ne renvoie pas d'erreur dans ce cas

Saisie d'une valeur par l'utilisateur

On peut utiliser indifféremment :

let nom = prompt("Quel est votre nom ?");
alert("Hello world " + nom);

ou

let nom = prompt("Quel est votre nom ?");
alert(`Hello world ${nom}`);

Attention toutefois avec la deuxième structure (template litteral) qui n'utilise plus des guillemets classiques mais des `.

BON A SAVOIR

Attention, la valeur saisie au clavier est systématiquement de type String. Il convient donc de la transformer en nombre le cas échéant.

Remarquons que les instructions prompt() et alert() ne sont pas supportées par Node.js

Template literal

JavaScript permet d'inclure des expressions dans une chaîne de caractères appelées template literal. Pour cela, il faut délimiter la chaîne avec ` à la place de " et utiliser ${expression}.

let nom = "Fabien";
console.log(`Je m'appelle ${nom}`);

Les listes

Nous utiliserons la dénomination suivante, qui est un abus de langage. Nous appelerons liste une structure de données dont la taille peut être amenée à changer et tableau une structure de taille fixe. Dans ce dernier cas, les éléments d'un tableau ne sont pas censés être modifiables.

On peut créer une liste (ou un tableau) de la manière suivante :

let tab = [element1, element2, element3];

L'indice du 1er élément d'une liste est 0.

Parcourir une liste

On peut toujours utiliser la technique classique suivante :

for(let i=0; i&l tab.length; i++){
    console.log(tab[i]);
  }

mais il est inutile de demander à l'interpréteur d'évaluer systématiquement tab.length. On préfèrera donc utiliser l'instruction suivante qui incorpore une nouvelle variable dans le scope des éléments à parcourir :

for(let i=0, c=tab.length; i<c; i++){
    console.log(tab[i]);
  }

On peut également utiliser l'instruction forEach comme dans d'autres langages. Dans ce cas, on utilise une fonction lambda, qu'on peut écrire de deux façons différentes :

tab.forEach(function(value){
    console.log(value);
    });

ou

tab.forEach(value => {console.log(value);});

Il est également possible d'utiliser le mot-clé in, mais attention dans ce cas, contrairement à Python, on n'itère pas les éléments de la liste les uns après les autres, mais on crée un indice semblable à i, pour lequel il n'est pas nécessaire de définir de valeur de départ et de valeur de fin. Afin de mieux comprendre, analysez l'exemple ci-dessous :

let prenoms = ["Alexandre", "Sarah", "Pierre"]

for (let i in prenoms) {
   console.log(prenoms[i]);
}

qui affiche :
Alexandre
Sarah
Pierre.

Si on utilisait maintenant la structure classique telle qu'elle existe en Python, il faudrait écrire 

let prenoms = ["Alexandre", "Sarah", "Pierre"]

  for (let elt in prenoms) {
     console.log(elt);
  }

qui affiche
0
1
2.

Dans ce cas, elt n'est pas un élément de la liste prenoms, mais bien un indice créé en fonction de la taille de cette liste. Attention donc à l'utilisation de in en JavaScript, qui se comporte différemment de ce qu'on peut penser.

En fait, l'équivalent de for ... in en Python, se traite en JavaScript avec une boucle for ... of, comme dans l'exemple ci-dessous :

for (let elt of prenoms){
    console.log(elt);
  }

qui affiche bien :
Alexandre
Sarah
Pierre.

Particularités des listes en JavaScript

En JavaScript on peut stocker des éléments de différents types dans une liste.

let tab = ["Antoine", 12, false];

En JavaScript, l'appel à un indice ne figurant pas dans le tableau ne conduit pas à une exception du type IndexOutOfRange mais renvoie la valeur undefined.

Copie de liste

On peut copier la liste L dans une nouvelle liste appelée newL en utilisant l'instruction :

let newL = [L];

Attention, en JavaScript les listes sont copiées par référence et non par valeur comme pour les types primitifs (string, number, bool). De ce fait, toute modification de L entraîne une modification de newL et inversement.

Les dictionnaires

Ils se construisent avec des accolades à la place des crochets et des virgules entre les éléments. Ils sont parfois appelés tableaux associatifs.

let dico ={
    nom: "Antoine",
    prenom: "Hervé"
  };

Pour accéder à un élément du dictionnaire, il ne faut plus appeler son indice qui n'existe pas mais appeler la clé de la façon suivante :

dico.nom

On remarque donc qu'un dictionnaire n'est autre qu'un objet dans lequel les clés sont des attributs qui possèdent des valeurs.

Dans certains cas, on préfèrera appeler l'élément du dictionnaire de la manière suivante :

dico["nom"]

Pour ajouter un élément à la fin d'un dictionnaire, il suffit de créer cet élément, car il n'existe pas de fin pour un dictionnaire, qui est structuré comme un ensemble non ordonné :

dico["lieuNaissance"] = "Lausanne";

ou encore

dico.lieuNaissance = "Lausanne";

Parcourir un dictionnaire

for(let cle in dico){
      console.log(`${cle} : ${dico[cle]}`);
    }

Les fonctions

La déclaration d'une fonction se réalise de la manière suivante :

function nomFonction(parametre){
  instructions;
  return valeur;
}

BON A SAVOIR

Attention, en JavaScript le type de retour de la fonction et le type des arguments ne sont pas spécifiés lors de la déclaration. Il est donc impossible de surcharger une fonction en JavaScript.

L'appel de la fonction se fait de la façon suivante :

let valeur = nomFonction(12);

Lorsqu'il n'y a pas de valeur de retour, on parle de procédure :

function nomProcedure(parametre){
  instructions;
}

L'appel de la procédure se fait de la façon suivante :

nomProcedure("Herve");

Documenter une fonction avec des docstring

Il est possible d'utiliser la structure suivante pour les docstring :

/**
 * Cette fonction prend un nom en parametre et renvoie un pseudo. 
 * 
 * @param {string} nom: le nom de l'utilisateur
 *
 * @return {string} pseudo: le pseudo renvoye
 */

Fonction lambda

Le principe de fonction lanbda (anonyme) utilise toujours le mot-clé function mais ne spécifie aucun nom pour la fonction créée.

let maVariable = function(parametre){
  let message = "Bonjour " + parametre;
  return message;
}

console.log(maVariable("Hervé"));

qu'on peut également écrire :

let maVariable = (parametre) =>{
  let message = "Bonjour " + parametre;
  return message;
}

console.log(maVariable("Hervé"));

On remarque qu'une fonction lambda utilise une construction particuliére et transforme la variable maVariable en fonction.

Programmation objet

La création d'une classe requiert les mot-clés class et constructor().

class MaClasse{
    
    constructor(nom, prenom){
      this.nom = nom;
      this.prenom = prenom;
    }
  
    uneMethode(parametre){
      instructions;
    }
  }

L'instanciation d'un objet se fait de façon classique :

let monObjet = new MaClasse("Charles", "Hervé");

Il existe également des méthodes statiques qui ne sont pas définies pour un objet mais pour une classe toute entière. Il n'est donc pas nécessaire de créer un objet pour les utiliser. Par exemple 

class MaClasse{

    constructor(nom, prenom){
      this.nom = nom;
      this.prenom = prenom;
    }

    static uneMethodeStatique(parametre){
      instructions;
    }
}

let monObjet = new MaClasse("Charles", "Hervé");
monObjet.uneMethode(parametre)
maClasse.uneMethodeStatique(parametre)

Particularité de JavaScript

JavaScript utilise des prototypes qui sont des liens vers d'autres objets pris en référence.

let monObjet = new MaClasse(nom, prenom);
let unAutreObjet = Object.create(monObjet);

Dans l'exemple ci-dessus, l'objet unAutreObjet utilise le modèle de monObjet pour se construire qui lui-même est une instanciation de maClasse.

La gestion des exceptions

On distingue souvent 3 catégories d'erreurs : les erreurs de syntaxe, les erreurs de logique et les erreurs d'exécution.

En JavaScript il est possible de traiter activement une erreur potentielle à l'aide d'une condition if ... else. Il est également possible de gérer les exceptions à l'aide de l'instruction :

try{
  // code susceptible de fournir un message d'erreur
}catch (error){
  // instructions à suivre en cas d'erreur
}

Ici error peut être remplacé par le libellé exact du code d'erreur.
