La structure d'un projet Python

Jusqu'à présent, nous avons appris à construire notre programme dans une seule page. La notion de modularité offre une architecture différente pour notre projet. Celui-ci sera plus facile à comprendre, à modifier puis éventuellement à agrandir.

Un module en Python est en fait un fichier, une librairie ou un package. Pour bien comprendre la structure d'un projet, il est nécessaire que celle-ci suive un protocole connu et efficace, où chaque module se trouve à un emplacement prédéfini, comme par exemple le design pattern MVT Model View Template de Django. Cela permet par exemple à tout développeur qui reprend le travail d'un autre d'être immédiatement efficace, ou encore de travailler plus facilement en groupe, puisque chacun sait dans quel dossier il doit travailler. Par ailleurs, cela limite les conflits lors des commits (enregistrements sur un dépôt) de modifications de code, puisque chaque développeur n'a travaillé que sur sa partie de code.

La structure de modules la plus simple est la suivante :

  • un module principal souvent appelé __main__.py
  • un module contenant les fonctions nécessaires au programme
  • un module contenant la base de données, par exemple au format .JSON

Nous avons déjà travaillé sur des bases de données distantes comme .CSV ou .JSON. Nous n'allons donc pas revenir dessus. En revanche, le module principal sera construit comme dans l'exemple ci-dessous où le code appelle une fonction bonjour() :

import fonctions as f 

if __name__ == "__main__":
    print(f.bonjour("rachel"))

On constate qu'il faut importer le module fonctions contenant les fonctions du projet, comme n'importe quelle librairie. Attention, fonctions est considéré comme un module et non plus comme un fichier Python. Il ne faut donc pas lui donner l'extension .PY au moment de l'importation.

Par la suite, toutes les fonctions importées devront être précédées du préfixe f qu'on vient de définir comme représentant le module fonctions. f.bonjour() est alors reconnue comme si la fonction avait été définie dans notre fichier.

On constate également qu'il faut utiliser l'instruction if __name__ == "__main__" qui empêche les modules autres que le module principal d'être exécutées au moment où ils sont importées. C'est en quelque sorte une sécurité pour s'assurer que seul le module principal s'exécute, les autres modules étant réduits à des supports pour le module principal.

Le fichier fonctions.py (ou module fonctions) contient bien évidemment la définition de la fonction bonjour() :

def bonjour(nom):
    return "hello " + nom

Rappels sur l'importation d'une librairie ou d'un module

Il est parfois très utile de savoir importer une bibliothèque, par exemple lorsqu'on veut introduire un paramètre de fatalité dans certains jeux, lorsqu'on veut utiliser la fonction racine carrée ou sinus en mathématiques, lorsqu'on veut créer un modèle spécifique en intelligence artificielle. Une bibliothèque est un ensemble de fonctionnalités et de méthodes qui permettent de fournir certains résultats aux questions que d'autres développeurs se sont déjà posées. Schématiquement, il s'agit d'un ensemble de fonctions qui vous permettent d'éviter de reconstruire ce qui existe déjà.

Les bibliothèques représentent donc un gain de temps précieux pour vous, et il est astucieux de s'en servir à profusion, d'autant plus qu'il existe probablement une bibliothèque répondant aux questions de vous vous posez lorsque vous créez un programme.

Vous devez vous demander "pourquoi faut-il les installées si elles sont si importantes ?" Tout simplement pour ne pas encombrer la mémoire occupée par votre programme avec des bibliothèques inutiles. Par exemple, à quoi cela sert-il de pouvoir utiliser la fonction tangente lorsqu'on crée un jeu vidéo ? Dans ce cas la bibliothèque math ne semble pas très utile.

A RETENIR ...

Le fait d'importer certains modules, uniquement lorsque nous en avons besoin, permet au langage de préserver de la mémoire.

Apprenons maintenant à importer une bibliothèque. Les bibliothèques Python sont téléchargées et installées sur votre ordinateur automatiquement par votre environnement de développement, ou manuellement si vous souhaitez les installer vous-même à un emplacement spécifique.

Importer une bibliothèque ne consiste pas à la télécharger sur Internet et à l'installer, mais à la récupérer à partir de son emplacement d'installation afin de la rendre disponible pour votre programme. Pour cela, rien de plus simple, il suffit d'utiliser en début de programme l'instruction import.

Il existe 4 façons de procéder, toutes valides. Prenons pour exemple la bibliothèque math afin de calculer \(\sqrt{3}\). Dans le premier cas, on importe toute la bibliothèque et on doit l'appeler devant la fonction sqrt() à chaque fois qu'on l'utilise.

import math

print(math.sqrt(3))

Dans le deuxième cas, on importe la bibliothèque math en lui donnant un pseudo plus rapide à écrire, qu'on doit utiliser devant la fonction sqrt() à chaque fois qu'on l'utilise.

import math as m

print(m.sqrt(3))

Dans le troisième cas, on importe la bibliothèque math sans lui donner de pseudo, mais il n'est pas nécessaire de l'appeler devant la fonction sqrt() à chaque fois qu'on l'utilise. C'est très pratique mais cela peut entraîner des confusions si on utilise deux fonctions portant le même nom mais appartenant à des bibliothèques différentes.

from math import *

print(sqrt(3))

Dans le quatrième cas, on importe uniquement la fonction sqrt() de la bibliothèque math et on ne peut donc utiliser aucune autre fonction de cette bibliothèque. Il n'est pas non plus nécessaire d'utiliser le nom de la bibliothèque devant la fonction sqrt().

from math import sqrt

print(sqrt(3))

Python est un langage de programmation permettant des calculs scientifiques poussés et des représentations graphiques élaborées, grâce à l'utilisation de bibliothèques conçues à cet effet.

En mathématiques, nous utiliserons souvent le module math pour accéder à certaines fonctions, et le module Random pour fournir des résultats aléatoires en probabilité. Nous utiliserons plus rarement le module numpy pour accéder à certaines fonctionnalités de calcul.

Nous commencerons donc souvent notre code par :

from math import *
from Random import *
