L'indentation

L'indentation est le fait de décaler le code vers la droite (d'une tabulation ou de 4 espaces). C'est essentiel en Python qui n'utilise par d'accolades et choisit de commencer un bloc de code par une indentation et de rester dans ce bloc tant que le code est indenté. Par exemple dans le code suivant :

nom = "Alice"
if nom == "Alice":
    print("Bonjour Alice.")
    print("Comment allez-vous aujourd'hui ?")
print("Aujourd'hui c'est journée portes ouvertes.")

Bonjour Alice. Comment allez-vous aujourd'hui ? ne s'affichera que si le nom vaut "Alice". Les deux phrases font partie du même bloc d'instructions. En revanche, Aujourd'hui c'est journée portes ouvertes. s'affichera tout le temps, même si nom ne vaut pas "Alice", car cette instruction est indépendante de l'instruction if nom == "Alice":.

Les booléens

nom = "Alice"
if nom == "Alice":
    print("Bonjour Alice")
else:
    print("Désolé, accès refusé.")

Les conditions sont intimement liées aux booléens (True ou False).

En effet, il faut comprendre que dans l'instruction if nom == "Alice":, le mot-clé if permet d'orienter le programme dans une certaine direction lorsque la condition nom == "Alice" est vraie, c'est à dire vaut True. Dans ce cas, le programme poursuit son exécution avec le bloc qui suit. Si le nom n'est pas Alice, alors la condition nom == "Alice" est fausse, donc équivalente à False et le programme décide de sauter le bloc relatif au mot-clé if et exécute le bloc relatif au mot else:.

A vous de jouer : pour se convaincre du rôle des booléens dans les conditions, il suffit d'exécuter le code suivant

nom = "Albert"
print(nom == "Albert")  # affiche True
print(nom == "Bob")  # affiche False
print(12 < 5)  # affiche False
print(7 < 10)  # affiche True

Les opérateurs utilisés

La condition nom == "Alice" se traduit par True ou False, mais il existe d'autres opérateurs de comparaison que ==. Les principaux opérateurs utilisés sont les suivants :

opérateursignification
a == bTrue si a vaut b
a != bTrue si a différent de b
a < bTrue si a inférieur à b
a > bTrue si a supérieur à b
a <= bTrue si a inférieur ou égal à b

Les opérateurs and / or

L'opérateur and nécessite que plusieurs conditions soient vraies pour que la condition globale soit vraie.

L'opérateur or nécessite qu'une au moins des conditions soit vraie pour que la condition globale soit vraie.

Exécuter le code suivant pour bien comprendre le fonctionnement des opérateurs and et or :

1 | a = 3
2 | b = 5
3 |
4 | if a > 4 and b > 4:
5 |    print("a et b sont tous les deux supérieurs à 4")
6 |
7 | if a > 4 or b > 4:
8 |    print("a ou b est supérieur à 4")

La condition a > 4 and b > 4 nécessite que a et b soient tous les deux supérieurs à 4. Cette condition n'est pas vérifiée et le programme n'entre pas dans le bloc de la ligne 5.

La condition a > 4 or b > 4 nécessite qu'une seule des deux variables a ou b soit supérieure à 4. Cette condition est donc vérifiée et le programme entre dans le bloc de la ligne 8.

Comprendre les messages d'erreur

Il est très important de savoir analyser les messages d'erreur pour corriger les bugs et progresser rapidement, car l'interpréteur Python fournit une aide précieuse et propose même parfois des correctifs.

Il existe différents types d'erreur :

  • les erreurs de syntaxe. Très faciles à corriger, il s'agit en général d'une mauvaise structure de code qui saute rapidement aux yeux avec l'usage,
  • les erreurs de logique. Elles sont souvent difficiles à trouver et il faut parfois recourir à des fonctions de tests et de validation pour savoir d'où provient l'erreur.

Analysons les messages d'erreur les plus fréquents proposés par l'interpréteur. Chaque message d'erreur commence par :

Traceback (most recent call last):

ce qui signifie qu'à la fin du message apparaît le numéro de la ligne de code où l'erreur a été déclenchée, et qu'au début du message le numéro de ligne de code à l'origine de l'erreur. C'est donc à cette ligne (ou parfois la suivante) qu'il faut effectuer les modifications.

name is not defined

Traceback (most recent call last):
    File "", line 4, in 
NameError: name 'prenom' is not defined

Cela signifie que la variable appelée prenom n'est pas définie à la ligne 4. Le code l'utilise comme si elle existait mais aucune valeur ne lui a été attribuée.

Solution : il faut donc donner une valeur à la variable prenom avant la ligne 4.

SyntaxError

Traceback (most recent call last):
    File "", line 2
if prenom == "Antoine"
                          ^
SyntaxError: expected ':'

Cela signifie que la syntaxe est mauvaise, et la petite flèche ^ montre l'endroit exact où manque le ':' à la ligne 2.

Solution : il faut rajouter ':' après "Antoine"

IndentationError

Traceback (most recent call last):
    File "", line 3
print("hello")
^^^^^
IndentationError: expected an indented block after 'if' statement on line 2

Cela signifie qu'après la condition if il est nécessaire de décaler les instructions à exécuter vers la droite. Le décalage est attendu à la ligne 3 au niveau des signes ^^^^^

Solution : indenter le print() d'une tabulation vers la droite.

index out of range

Traceback (most recent call last):
    File "", line 3, in 
IndexError: string index out of range

Cela signifie qu'une boucle parcourt un indice qui n'existe pas à la ligne 3. Il faut donc vérifier le premier et le dernier éléments de la boucle dont l'un des deux n'est sûrement pas défini.

Solution : il faut changer l'indice de début de boucle ou l'indice de fin de boucle.

La prime d’assurance de base est de 500 CHF.   • pour 1 accident déclaré durant les 5 dernières années, la surprime est de 50 CHF
  • pour 2 accidents déclarés durant les 5 dernières années, la surprime est de 125 CHF
  • pour 3 accidents déclarés durant les 5 dernières années, la surprime est de 225 CHF
  • pour 4 accidents déclarés durant les 5 dernières années, la surprime est de 375 CHF
  • pour 5 accidents déclarés durant les 5 dernières années, la surprime est de 575 CHF

else:
    # on compare l'age du client avec 25
    if age < 25:
        prime_assurance = prime_assurance + 100

    # on rajoute la surprime en fonction de son nombre d'accidents
    if nbre_accidents == 1:
        prime_assurance = prime_assurance + 50
    elif nbre_accidents == 2:
        prime_assurance = prime_assurance + 125
    elif nbre_accidents == 3:
        prime_assurance = prime_assurance + 225
    elif nbre_accidents == 4:
        prime_assurance = prime_assurance + 375
    elif nbre_accidents == 5:
        prime_assurance = prime_assurance + 575

    # on affiche le montant à payer
    print(prime_assurance)
