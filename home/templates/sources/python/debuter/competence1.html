La structure des blocs de code Python

Chaque programme est composé d'instructions, qui sont comme des phrases avec leur propre syntaxe, leur orthographe et leur grammaire. Contrairement à la langue française, le nombre de mots est très limité, et la syntaxe est toujours la même. Elle peut paraître compliquée au début mais le meilleur moyen de bien la connaître est de pratiquer en s'aidant des commentaires de l'ordinateur pour effectuer les corrections. Il faut par ailleurs toujours garder à l'esprit que l'ordinateur exécute à la lettre ce qu'on lui demande de faire, ligne après ligne.

Les phrases composent des paragraphes, qui eux-même composent des chapitres qui composent un livre. En suivant cette analogie, une ligne de code serait une phrase, un bloc ou ensemble d'instructions regroupées entre elles serait un paragaphe, un module serait un chapitre et un projet serait un livre.

Le langage Python a la particularité de ne pas séparer les blocs de code par des accolades. Il utilise plutôt un niveau de décalage à droite (appelé indentation) représentatif de chaque bloc.

Par exemple, au lieu d'avoir un code structuré ainsi :

{
instruction du bloc 1
instruction du bloc 1
{
instruction du bloc 2
instruction du bloc 2
}
instruction du bloc 1
}

le même code en Python serait

instruction du bloc 1
instruction du bloc 1
    instruction du bloc 2
    instruction du bloc 2
instruction du bloc 1

On remarque tout de suite que le code Python est plus court et plus lisible. En revanche, il est impératif de penser à décaler les instructions du bloc 2, sinon elles appartiendraient au bloc 1 et cela changerait le code.

Par ailleurs, la fin d'une ligne symbolise la fin d'une instruction. Il n'est pas nécessaire en Python d'utiliser des points-virgules pour symboliser la fin d'une instruction.

Il est possible et même très important d'écrire des commentaires dans un programme. Ces portions de code ne sont pas exécutées par l'interpréteur Python et ont pour but de faciliter la compréhension du code, pour le lecteur du code comme pour le développeur (car parfois les codes sont compliqués). Les commentaires d'une ligne commencent toujours par le symbole #, alors que les commentaires de plusieurs lignes sont entourés des symboles """, comme dans l'exemple ci-dessous.

# ceci est un commentaire d'une ligne

"""
ceci est un commentaire
sur plusieurs lignes
"""

Il faut prendre l'habitude d'écrire les commentaires en anglais car Python est écrit en anglais. A défaut, il faut écrire en français sans accent, ou alors rajouter dès la première ligne de code l'instruction suivante qui indique à Python que les accents et caractères spéciaux sont autorisés :

# -*- coding: utf-8 -*-

Découvrir les variables

Une variable est un peu comme un carton de rangement. On a besoin d'un nom pour savoir ce qu'il contient, et bien sûr on doit pouvoir accéder à son contenu quand on en a besoin. On doit même pouvoir retirer une affaire du carton, rajouter une affaire ou en modifier totalement le contenu. Afin de pouvoir le retrouver facilement, on a besoin de savoir à quelle famille il appartient, par exemple de taille moyenne et de couleur rouge. Il est également important de connaître son emplacement dans la maison lorsqu'on a besoin d'y accéder. Une variable est construite suivant le même principe qu'un carton, et dispose des mêmes caractéristiques : nom, contenu, emplacement, famille.

Une variable permet de stocker une donnée du code. Par exemple dans un jeu, on a besoin de garder en mémoire le score du joueur, sur un site internet on a besoin de garder en mémoire le pseudo de l'utilisateur lorsqu'il est identifié. On comprend bien que les variables sont les briques les plus essentielles de tout programme.

Une variable est constituée d'un nom, d'une valeur, d'un emplacement en mémoire et d'un type. En Python, on ne s'occupera que du nom de la variable, de sa valeur et de son type.

A noter que le nom d'une variable ne peut pas commencer par un chiffre, ne peut pas contenir d'espace et ne peut pas être un mot réservé, c'est à dire une instruction Python. Voici un premier exemple où le nom de la variable est age :

age = 18

On utilise le signe "=" pour donner la valeur 18 à la variable appelée age. On peut maintenant changer la valeur de age autant de fois que l'on souhaite. Celle-ci est stockée en mémoire sous le nom age. C'est parce qu'on peut en changer la valeur autant de fois que l'on veut, qu'on parle de "variable".

⚡ Attention, en mathématiques le signe "=" est réversible, c'est à dire qu'on peut l'utiliser dans les deux sens et on peut également résoudre une équation pour obtenir la valeur d'une inconnue x par exemple. En informatique, c'est beaucoup plus simple, le signe "=" signifie uniquement qu'on donne une valeur à une variable. Il n'y a pas de notion d'égalité. Pour mieux comprendre, analysons le code ci-dessous :

x = 12 # x prend la valeur 12
y = x
x = 44

Un programme se lit et s'exécute ligne par ligne. Ici, x prend la valeur 12, puis y prend la valeur de x c'est à dire 12, finalement x prend la valeur 44. C'est juste un exemple sans intérêt, si ce n'est pour bien comprendre qu'à la fin x vaut 44 et y vaut 12 alors qu'il était écrit y = x. Les différents types de variables

Chaque variable appartient à une famille particulière relative à la valeur de cette variable :

  • les nombres entiers sont stockés dans des variables de type int pour integer
  • les nombres réels sont stockés dans des variables de type float pour nombres à virgules flottantes
  • les textes (chaînes de caractères) sont stockés dans des variables de type str pour string
  • les valeurs True et False sont stockées dans des variables de type bool pour booléens.

A noter que la place qu'occupe une variable en mémoire dépend fortement du type de cette variable. C'est la raison pour laquelle il est parfois préférable de travailler avec un type de variable plutôt qu'un autre.

On peut afficher le type d'une variable avec l'instruction type() et le nom de la variable. Par exemple : 

print(type(age))

affichera int.

Les conventions de notation

Les noms de variables doivent avoir un sens et s'écrivent très souvent d'une des deux manières suivantes :

nomUtilisateur = "Antoine"
nom_utilisateur = "Antoine"

Les constantes (qui ne varient donc pas) portent un nom qui s'écrit en majuscules, et sont considérées en Python comme étant un cas particulier de variables. Par exemple :

PI = 3.14

Rappel, le nom d'une variable ne commence pas par un chiffre et ne peut pas être un mot-clé du langage Python, comme par exemple print, if, else, while, def, int, str, float, bool, for, range, not, chr, ord, list, ... Par ailleurs, les virgules sont représentées par des points en anglais et donc également en Python, comme dans l'exemple précédent de PI.

Les principaux opérateurs

Les opérateurs servent à modifier la valeur des variables. Pour les nombres entiers (int) et les nombres réels (float), on peut utiliser les opérations suivantes :

+ pour l'addition

L'instruction ci-dessous donne la valeur 8 à la variable x.

x = 5 + 3

- pour la soustraction

L'instruction ci-dessous donne la valeur 3 à la variable x.

x = 5 - 2

* pour la multiplication

L'instruction ci-dessous donne la valeur 35 à la variable x.

x = 5 * 7

/ pour la division

L'instruction ci-dessous donne la valeur 1.4 à la variable x.

x = 7 / 5

** pour l'exposant

L'instruction ci-dessous donne la valeur 49 à la variable x.

x = 7 ** 2

// pour le quotient d'une division

Il est également possible d'effectuer une division euclidienne. Par exemple, on sait que \(7 = 1 \times 5 + 2\).

L'instruction ci-dessous donne la valeur 1 à la variable x.

x = 7 // 5

Attention la division décimale de 7 par 5 donne 1.4, mais la division euclidienne de 7 par 5 donne 1 fois 5 plus 2. Ainsi le quotient vaut 1 et le reste de division vaut 2, ce qui explique le résultat obtenu.

% pour le reste de division

L'instruction ci-dessous donne la valeur 2 à la variable x. On utilise encore une fois la division euclidienne. Celle-ci permet de mettre en valeur l'opérateur modulo % pour savoir si un nombre est multiple d'un autre, ou si une séquence de caractères peut être regroupée en blocs de 8 éléments.

x = 7 % 5

Manipuler les variables

Inverser deux variables

En Python, on peut définir plusieurs variables sur une même ligne, en les séparant par une virgule. Par exemple :

nom, prenom = "Gulliver", "Cedric"

On veut maintenant inverser les valeurs de deux variables. En fait, ce n'est pas si simple. Pour s'en rendre compte, il suffit d'imaginer deux fichiers Word appelés monFichier1.docx et monFichier2.docx. Si on veut changer le nom de monFichier1.docx en monFichier2.docx, le système d'exploitation de la machine refuse cette modification car on ne peut pas avoir deux fichiers du même nom dans un même dossier.

Si on bouge monFichier2.docx dans un autre dossier, qu'on renomme monFichier1.docx, cela va marcher mais on ne pourra plus faire revenir monFichier2.docx dans son dossier initial puisqu'on aurait dans ce cas deux fichiers du même nom.

Il faut en fait utiliser un nom de fichier temporaire pour monFichier1.docx, par exemple l'appeler temp.docx. On peut ensuite renommer monFichier2.docx en monFichier1.docx et temp.docx en monFichier2.docx.

Effectuer des calculs avec des variables

On peut calculer la valeur d'une variable en utilisant celle d'autres variables. Par exemple si on veut trouver la hauteur d'un muret composé de pots surmontés d'arbustes, on peut effectuer le calcul suivant

hauteur_pots = 0.6
hauteur_arbustes = 1.2
hauteur_muret = hauteur_pots + hauteur_arbustes

L'avantage d'utiliser des variables plutôt que des valeurs directement est le même que celui du calcul littéral en mathématiques. Cela rend le calcul plus facile lorsqu'on doit changer les valeurs initiales.

A toi de jouer : écris un script qui calcule la vitesse moyenne d'un cycliste, connaissant la distance de montée de 16 km, la distance de descente de 51 km, le temps de montée de 1,2 heures et le temps de descente de 1,5 heures.
Indication : la vitesse moyenne se calcule en faisant la moyenne de la vitesse de montée et de la vitesse de descente. Une vitesse est le quotient d'une distance par un temps, d'où l'unité km/h.

On peut tout à fait utiliser Python comme une simple calculatrice. Dans ce cas, il suffit d'écrire les lignes suivantes pour notre programme :

vitesse_moyenne = (16 / 1.2 + 51 / 1.5) / 2
print(vitesse_moyenne)

Toutefois, il est souvent astucieux de créer plusieurs variables et de détailler nos calculs afin de pouvoir réutiliser des portions de code ou les variables dans d'autres parties du programme. On pourrait donc écrire :

distance_montee = 16 # en km
temps_montee = 1.2 # en heures
distance_descente = 51
temps_descente = 1.5

vitesse_montee = distance_montee / temps_montee
vitesse_descente = distance_descente / temps_descente

vitesse_moyenne = (vitesse_montee + vitesse_descente) / 2
print(vitesse_moyenne)

Il peut parfois être utile de connaître la durée d'exécution d'une instruction. Pour cela, on utilise la bibliothèque time. L'instruction time() renvoie le temps en secondes depuis le 1er janvier 1970 (big bang de l'informatique). L'instruction sleep() permet de suspendre l'exécution d'une partie d'un programme pendant un certain nombre de secondes. Le code suivant permet par exemple de se rendre compte du temps nécessaire à Python pour compter jusqu'à 100 000 :

import time

print(time.time())

compteur = 0
for i in range(100000):
    compteur = compteur + 1

print(time.time())

Afficher les variables à l'écran

Nous allons voir comment afficher les valeurs de plusieurs variables, ou la valeur d'une variable au sein d'un texte.

nom = "Parker"
prenom = "Peter"
print(prenom + nom) # affiche PeterParker
print(prenom + " - " + nom) # affiche Peter - Parker
print(prenom + "\t" + nom) # affiche Peter   Parker séparés par une tabulation
print(prenom + "\n" + nom) # affiche Peter Parker l'un en dessous de l'autre

On remarque donc que l'instruction print() saute automatiquement la ligne, une fois le travail effectué. Elle correspond en fait au println() qu'on retrouve dans d'autres langages.

Affichage respectant un format

On a vu ensemble qu'on pouvait afficher la valeur d'une variable au milieu d'un texte, en construisant un texte qui contient cette variable. Par exemple le code suivant :

prenom = "Antoine"
date_naissance = "17-02-2008
print(prenom + " est né le " + date_naissance)

affichera le résultat suivant :

Antoine est né le 17-02-2008

Il existe une façon plus efficace d'afficher le même résultat en utilisant l'instruction print(f) :

prenom = "Antoine"
date_naissance = "17-02-2008
print(f"{prenom} est né le {date_naissance}")

Importer une bibliothèque

Il est parfois très utile de savoir importer une bibliothèque, par exemple lorsqu'on veut introduire un paramètre de fatalité dans certains jeux, lorsqu'on veut utiliser la fonction racine carrée ou sinus en mathématiques, lorsqu'on veut créer un modèle spécifique en intelligence artificielle. Une bibliothèque est un ensemble de fonctionnalités et de méthodes qui permettent de fournir certains résultats aux questions que d'autres développeurs se sont déjà posées. Schématiquement, il s'agit d'un ensemble de fonctions qui vous permettent d'éviter de reconstruire ce qui existe déjà.

Les bibliothèques représentent donc un gain de temps précieux pour vous, et il est astucieux de s'en servir à profusion, d'autant plus qu'il existe probablement une bibliothèque répondant aux questions de vous vous posez lorsque vous créez un programme.

Vous devez vous demander "pourquoi faut-il les installées si elles sont si importantes ?" Tout simplement pour ne pas encombrer la mémoire occupée par votre programme avec des bibliothèques inutiles. Par exemple, à quoi cela sert-il de pouvoir utiliser la fonction tangente lorsqu'on crée un jeu vidéo ? Dans ce cas la bibliothèque math ne semble pas très utile.

A RETENIR ...

Le fait d'importer certains modules, uniquement lorsque nous en avons besoin, permet au langage de préserver de la mémoire.

Apprenons maintenant à importer une bibliothèque. Les bibliothèques Python sont téléchargées et installées sur votre ordinateur automatiquement par votre environnement de développement, ou manuellement si vous souhaitez les installer vous-même à un emplacement spécifique.

Importer une bibliothèque ne consiste pas à la télécharger sur Internet et à l'installer, mais à la récupérer à partir de son emplacement d'installation afin de la rendre disponible pour votre programme. Pour cela, rien de plus simple, il suffit d'utiliser en début de programme l'instruction import.

Il existe 4 façons de procéder, toutes valides. Prenons pour exemple la bibliothèque math afin de calculer \(\sqrt{3}\). Dans le premier cas, on importe toute la bibliothèque et on doit l'appeler devant la fonction sqrt() à chaque fois qu'on l'utilise.

import math

print(math.sqrt(3))

Dans le deuxième cas, on importe la bibliothèque math en lui donnant un pseudo plus rapide à écrire, qu'on doit utiliser devant la fonction sqrt() à chaque fois qu'on l'utilise.

import math as m

print(m.sqrt(3))

Dans le troisième cas, on importe la bibliothèque math sans lui donner de pseudo, mais il n'est pas nécessaire de l'appeler devant la fonction sqrt() à chaque fois qu'on l'utilise. C'est très pratique mais cela peut entraîner des confusions si on utilise deux fonctions portant le même nom mais appartenant à des bibliothèques différentes.

from math import *

print(sqrt(3))

Dans le quatrième cas, on importe uniquement la fonction sqrt() de la bibliothèque math et on ne peut donc utiliser aucune autre fonction de cette bibliothèque. Il n'est pas non plus nécessaire d'utiliser le nom de la bibliothèque devant la fonction sqrt().

from math import sqrt

print(sqrt(3))

Python est un langage de programmation permettant des calculs scientifiques poussés et des représentations graphiques élaborées, grâce à l'utilisation de bibliothèques conçues à cet effet.

En mathématiques, nous utiliserons souvent le module math pour accéder à certaines fonctions, et le module Random pour fournir des résultats aléatoires en probabilité. Nous utiliserons plus rarement le module numpy pour accéder à certaines fonctionnalités de calcul.

Nous commencerons donc souvent notre code par :

from math import *
from Random import *
from numpy import *
