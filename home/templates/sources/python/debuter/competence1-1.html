Une variable est un peu comme un carton de rangement. On a besoin d'un nom pour savoir ce qu'il contient, et bien sûr on doit pouvoir accéder à son contenu quand on en a besoin. On doit même pouvoir retirer une affaire du carton, rajouter une affaire ou en modifier totalement le contenu. Afin d'utiliser la taille de carton la plus appropriée à son contenu, on a besoin de connaître le type de carton nécessaire. Il est également important de connaître son emplacement dans la maison lorsqu'on a besoin d'y accéder. Une variable est construite suivant le même principe qu'un carton, et dispose des mêmes caractéristiques : un nom, une valeur, un type et une adresse mémoire.

Une variable permet de stocker une donnée du code. Par exemple dans un jeu, on a besoin de garder en mémoire le score du joueur, sur un site internet on a besoin de garder en mémoire le pseudo de l'utilisateur lorsqu'il est identifié. On comprend bien que les variables sont les briques les plus essentielles au fonctionnement de nos programmes.

En Python, on ne s'occupera que du nom de la variable, de sa valeur et de son type. La gestion de la mémoire n'est pas du ressort du programmeur, contrairement au langage C. Elle est automatisée en Python par un garbage collector. Cela facilite le travail du développeur mais n'optimise pas l'utilisation de la mémoire et ralentit l'exécution du code.

Voici un premier exemple où le nom de la variable est age :

age = 18

On utilise le signe = pour donner la valeur 18 à la variable appelée age. On peut maintenant changer la valeur de age autant de fois que l'on souhaite. Celle-ci est stockée en mémoire sous le nom age. On parle de variable justement parce qu'on peut en changer la valeur autant de fois que nécessaire.

A toi de jouer : dans un jeu vidéo, le niveau peut varier de beginner à master. Donne la valeur beginner à une variable que tu appeleras niveau, puis la valeur 112 à une variable que tu appeleras score. Au cours du jeu le score du joueur monte de 40 points. Il atteint maintenant le niveau master. Modifie la valeur de ces variables dans le programme.

Exécuter Python à la console

Solution : 

niveau = "beginner"
score = 112

score  = 152
niveau = master

REMARQUABLE ...

En mathématiques le signe = est réversible, c'est à dire qu'on peut l'utiliser dans les deux sens et on peut également résoudre une équation pour obtenir la valeur d'une inconnue x par exemple. En informatique, c'est beaucoup plus simple, le signe = signifie uniquement qu'on donne une valeur à une variable. Il n'y a pas de notion d'égalité. Pour mieux comprendre, analysons le code ci-dessous :

x = 12
y = x
x = 44

Un programme se lit et s'exécute ligne par ligne. Dans ce cas,

  • x prend la valeur 12

  • y prend la valeur de x c'est à dire 12

  • x prend la valeur 44 et y conserve sa valeur 12. Bien qu'à la ligne précédent il était écrit y = x, les deux variables n'ont pas la même valeur à la fin du code. IL NE S'AGIT PAS D'UNE EGALITE MAIS DE L'AFFECTATION DE VALEURS A DES VARIABLES.

En résumé

  • les variables stockent des données en mémoire durant l'exécution d'un programme

  • une variable est constituée d'un nom, d'une valeur, d'un type et d'une adresse mémoire

  • on définit une variable en affectant une valeur à un nom grâce au signe =

  • le signe = ne symbolise pas une égalité, mais l'affectation d'une valeur à une variable

Comment nommer une variable

Le nom d'une variable ne peut pas commencer par un chiffre, ne peut pas contenir d'espace, ne peut pas contenir certains caractères spéciaux (#!.;()][/\<>+*) et ne peut pas être un mot-clé du langage Python comme par exemple print, if, else, while, def, int, str, float, bool, for, range, and, or, elif, input, match, not, chr, ord, list, ....

A toi de jouer : parmi les noms suivants, choisir ceux qu'on peut utiliser comme noms de variables (attention, on ne peut pas non plus utiliser de mot-clé Python comme nom de variable):

  • nA2rt5
  • compteur
  • 3noms
  • print
  • _nom
  • nom prenom
  • nom1
  • print4

On vient de voir que les noms de variables ne peuvent pas contenir d'espace. Pour cela, ils s'écrivent très souvent d'une des deux manières suivantes :

nomUtilisateur = "Antoine"
nom_utilisateur = "Antoine"

Les noms de variables doivent avoir un sens pour rendre le code plus lisible. Les deux exemples ci-dessous effectuent le même travail mais le 2ème code est bien plus compréhensible, même pour celui qui a créé le code lorsqu'il le relit quelques mois plus tard.

x = "23"
y = "7"
z = "2023"
jour = "23"
mois = "7"
annee = "2023"

Les constantes (qui ne varient donc pas) portent un nom qui s'écrit en majuscules, et sont considérées en Python comme étant un cas particulier de variables. Par exemple :

PI = 3.14
