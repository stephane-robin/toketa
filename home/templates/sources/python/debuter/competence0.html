Python est un langage interprété

Python est un langage de programmation dont la philosophie repose sur la simplicité du code et l'implémentation de bonnes pratiques. Rien n'est fait pour empêcher le programmeur de mener une action, même si celle-ci semble inappropriée. La structure du code en blocs indentés permet de meilleure lecture de code, et impose une structure commune et compréhensible pour tout codeur qui souhaiterait reprendre le programme.

Python est également un langage interprété. Cela signifie que le code source est traduit pas à pas par un logiciel appelé interpréteur, qui récupère des données d'entrée au fur et à mesure pour fournir des données de sortie. Le code source n'est pas transformé entièrement en code binaire avant d'être compilé, comme pour un langage compilé.

Le principal avantage est que le code source d'un langage interprété peut fonctionner sur tout ordinateur. Le principal inconvénient est qu'un langage interprété est plus lent qu'un langage compilé, qui peut réutiliser certaines parties de code préalablement compilées.

Grâce à ses bibliothèques performantes et régulièrement mises à jour, Python est particulièrement utilisé dans les environnements scientifiques, pour la manipulation de données et pour l'intelligence artificielle.

Bien choisir son outil pour exécuter du Python

On peut exécuter un programme Python en utilisant le Terminal d'un ordinateur (sous Linux et MacOS), mais il est plus pratique d'utiliser un éditeur de texte tel que Sublim Text ou NotePad++ qui colorient automatiquement le code en fonction du langage choisi et proposent une auto-complétion des instructions. Il est encore plus efficace d'utiliser un Environnement de Développement Intégré IDE tel que PyCharm ou JetBeans qui proposent en plus une recherche passive des erreurs de frappe, un debugger (aide pour corriger les bugs) et le paramétrage des outils de versioning (pour conserver l'historique du code).

Le debugger est un outil particulièrement intéressant, qui propose non seulement des commentaires relatifs aux erreurs de code, des solutions possibles, mais également la possibilité d'exécuter le code de point d'arrêt en point d'arrêt pour connaître l'état des variables du programme à chacune des lignes de son exécution, ces lignes ayant été préalablement définies par le développeur.

Un outil de versioning tel que Git est également très utile puisqu'il permet de conserver un historique des différentes versions de code, de revenir en arrière en choisissant une version plutôt qu'une autre si le code actuel n'est pas satisfaisant. Par ailleurs, il permet de travailler sur des copies de code plutôt que sur des originaux et de regrouper le travail de toute une équipe en évitant les conflits de code.

Python est souvent installé nativement sur votre ordinateur, mais sa version permet uniquement de faire fonctionner certaines applications de votre machine. Elle est indépendante de l'éditeur de texte ou de l'Environnement de Développement Intégré que vous utilisez pour exécuter du Python. De ce fait, l'utilisation du langage Python pour exécuter du code nécessite en général que l'utilisateur paramètre sur son ordinateur le chemin d'accès à l'interpréteur Python, car il y a de fortes chances pour que votre machine dispose de plusieurs versions de Python.

Lorsqu'on débute en Python, ce paramétrage peut s'avérer laborieux et décourageant. Il peut s'avérer pratique de se consacrer uniquement à la rédaction du code au lieu de passer du temps à installer et à configuration un Environnement de Développement fonctionnel. Pour cela, il existe des interpréteurs Python en ligne, présents sur Internet, qui exécutent votre code à partir d'un serveur distant ayant déjà installé et configuré Python.

En résumé, il est donc possible d'exécuter Python en utilisant l'un des outils suivants (du plus pratique au plus performant) :

  • sur un site internet proposant un interpréteur en ligne
  • à l'aide d'une commande sur votre Terminal et d'un éditeur de texte
  • à l'aide d'un environnement de développement intégré IDE

Nous allons détailler les 3 méthodes ci-dessus, mais il n'est nullement nécessaire de savoir toutes les utiliser., qui sont présentées en détail ci-dessous. Les interpréteurs en ligne sont très pratiques, mais ont principalement trois inconvénients. Exécuter son code à l'aide du Terminal et d'un éditeur de texte

Pour cela, il sera nécessaire d'installer une version récente de Python sur ton ordinateur, puis d'installer et de configurer un outil tel que Visual Studio Code, PyCharm, Sublim Text, Netbeans ou Atom. En revanche, si tu utilises déjà un IDE, c'est parfait, ne change rien.

Cette manière de procéder est plus complexe Elle requiert un peu d'expérience pour télécharger et installer un programme sur son ordinateur, sachant que pour chaque système d'exploitation l'installation est différente.

Il faut d'abord installer une version récente de Python sur sa machine. Pour cela, il faut se rendre sur le site officiel de Python, suivre la documentation, télécharger une version du langage et l'installer. Il existe de nombreux tutoriels expliquant comment installer une version de Python.

Il faut ensuite installer un éditeur de texte comme NotePad++ ou Sublim Text. Voici le lien vers le site officiel de Sublim Text.

Cette méthode présente deux inconvénients majeurs. Tout d'abord il faut télécharger et installer manuellement toutes les bibliothèques qui n'exitent pas nativement dans Python et qu'on souhaite utiliser. C'est le cas par exemple de matplolib ou de biopython. Ensuite, on ne dispose ni d'un correcteur de code intégré, ni d'un debuggeur. Toutefois, il peut être pratique d'exécuter un code directement au Terminal pour vérifier par exemple qu'un problème d'exécution n'est pas lié à l'environnement de développement qu'on utilise. Et puis cela permet de bien comprendre que Python ne fait pas partie de l'environnement de travail, mais qu'il est utilisé par cet environnement de travail.

L'exécution d'un programme est simple et rapide si les installations de Python et Sublim Text fonctionnent correctement.

A vous de jouer : par exemple, pour afficher "Hello world", il faut se placer dans un dossier comme Documents, creer un fichier appelé bonjour.py dans ce dossier, puis ouvrir ce fichier dans Sublim Text pour écrire le code suivant :

print("Hello world")

Ensuite, ouvrir le Terminal, se déplacer jusque dans le dossier Documents en exécutant l'instruction :

cd Documents
.

Pour exécuter une instruction au Terminal, il suffit d'appuyer sur la touche Entrée après avoir tapé l'instruction.

Finalement il ne reste plus qu'à exécuter le code suivant au Terminal :

python3 bonjour.py

La plus grosse difficulté et la principale source d'erreurs consiste à se placer dans le même dossier que le fichier bonjour.py avant de l'exécuter.

Si tout fonctionne normalement, tu dois voir s'afficher dans ton Terminal :

Hello world

On constate donc que l'éditeur de texte sert à écrire son code, et le Terminal demande à l'ordinateur d'exécuter le langage Python.

Découvrir Python avec un jeu de devinette

Voici le code d'un programme Python qui tire un nombre au hasard et demander au joueur de le deviner en 5 tentatives.

from random import randint

nbre_essais_max = 5
tentative = 1
min = 0
max = 50
choisi = randint(min, max)   
proposition = -1                      

print("Devine en " + str(nbre_essais_max) + " le nombre que j'ai choisi, entre " + str(min) + " et " + str(max))

while proposition != choisi and tentative <= nbre_essais_max:
    proposition = int(input("Ta proposition : "))
    if proposition < choisi:
        print("Trop petit")
    elif proposition > choisi:
        print("Trop grand")
    else:
        print("Bravo ! Tu as trouvé " + str(choisi) + " en " + str(tentative) + " essai(s)")
    tentative += 1
        
if tentative > nbre_essais_max and proposition != choisi :
    print("Perdu. J'avais choisi le nombre : " + str(choisi))

A toi de jouer : crée un fichier vierge à l'aide de Visual Studio Code. Copie-colle le programme ci-dessus dans ce nouveau fichier, donne un nom à ton fichier comme par exemple premierJeu.py, puis exécute le code. Attention, l'extension .PY est importante car elle indique à ton environnement de travail que le programme est écrit en Python.

  • Pour mieux comprendre ce que fait le programme, lit les lignes de codes une à une en essayant de deviner ce qu'elle fait.
  • Si on veut maintenant que le jeu se joue en 10 tentatives, que faut-il changer dans le code ?
  • Si on veut ensuite que le nombre à deviner soit compris entre 1 et 100, que doit-on changer dans le code ?
  • Si on veut finalement truquer le jeu pour que le joueur ne trouve jamais la solution. Comment pourrait-on changer le code ?

Découvrir Python avec une course d'escargots

Voici le code d'un programme Python qui fait courir trois escargots jusqu'à une ligne d'arrivée et affiche vainqueur lorsque le premier escargot a terminé.

from tkinter import *
from time import sleep
from random import randint

def demarrer():
    global x1, x2, x3
    y = 1000
    while x1 < 1040 and x2 < 1040 and x3 < 1040:
        nbre_hasard = randint(1,3)
        if nbre_hasard == 1:
            x1 += 10
            can.coords(esca1, x1, y1)
            y = y1
        elif nbre_hasard == 2:
            x2 += 10
            can.coords(esca2, x2, y2)
            y = y2
        else:
            x3 += 10
            can.coords(esca3, x3, y3)
            y = y3
        sleep(0.05)  
        can.update()
    can.coords(vainqueur, 500, y)


def reinit():
    global x1, x2, x3, y1, y2, y3
    can.coords(esca1, x1, y1)
    can.coords(esca2, x2, y2)
    can.coords(esca3, x3, y3)
    can.coords(vainqueur, 200, 800)

x1, y1  = 130, 75
x2, y2  = 130, 225
x3, y3  = 130, 375

fen = Tk()
fen.title("Course d'escargots")
can = Canvas(fen, width=1100, height=500, bg ='white')
can.pack(side=TOP, padx=5, pady=5)
b2 = Button(fen, text ='Départ de la course', width=15, command=demarrer)
b2.pack(side=RIGHT)

can.create_line(1080, 0, 1080, 500, width=5, fill="red")
can.create_line(250, 0, 250, 500, width=5, fill="green")
can.create_line(0, 150, 1100, 150, width=5, fill="black")
can.create_line(0, 300, 1100, 300, width=5, fill="black")
can.create_line(0, 450, 1100, 450, width=5, fill="black")

photo1 = PhotoImage(file ='~/Documents/ecg1.png')
esca1 = can.create_image(x1, y1, image=photo1)
photo2 = PhotoImage(file ='~/Documents/ecg2.png')
esca2 = can.create_image(x2, y2, image=photo2)
photo3 = PhotoImage(file ='~/Documents/ecg3.png')
esca3 = can.create_image(x3, y3, image=photo3)
photo4 = PhotoImage(file ='~/Documents/vainqueur.png')
vainqueur = can.create_image(200, 800, image=photo4) 

fen.mainloop()
fen.destroy()

A toi de jouer : crée un fichier vierge à l'aide de Visual Studio Code. Copie-colle le programme ci-dessus dans ce nouveau fichier, donne un nom à ton fichier comme par exemple escargots.py. Télécharge ensuite les images ecg1.png, ecg2.png, ecg3.png et vainqueur.png, puis place ces images dans le même dossier que ton fichier escargot.py. Exécute ensuite le code pour jouer.

Il est possible que ton programme rencontre des difficultés pour trouver l'adresse des images. Tu peux dans ce cas placer ton fichier escargots.py ainsi que les images dans le dossier Documents et modifier les adresses du type 'ecg1.png' en adresses du type '~/Documents/ecg1.png'
