Python est un langage interprété

Python est un langage de programmation généraliste qui permet de construire tous types de projets : études et gestion de bases de données, calculs scientifiques, graphismes, jeux vidéos, applications, sites Web, intelligence artificielle, ...

Les languages de programmation les plus populaires en juin 2023 sont (source index Tiobe):

  • Python
  • C
  • C++

Les languages de programmation les plus demandés sur les offres d'emploi en 2023 sont (source devjobscanner.com):

  • JavaScript
  • Python
  • Java

Tous indices confondus, on constate en fait une large domination de 4 langages : Python, C, C++ et Java.

Python n'est donc pas qu'un langage scolaire, mais un véritable langage polyvalent utilisé pour tous types de projets.

Python est un langage de programmation dont la philosophie repose sur la simplicité du code et l'implémentation de bonnes pratiques. Rien n'est fait pour empêcher le programmeur de mener une action, même si celle-ci semble inappropriée. La structure du code en blocs indentés permet de meilleure lecture de code, et impose une structure commune et compréhensible pour tout codeur qui souhaiterait reprendre le programme.

Python est également un langage interprété. Cela signifie que le code source est traduit pas à pas par un logiciel appelé interpréteur, qui récupère des données d'entrée au fur et à mesure pour fournir des données de sortie. Le code source n'est pas transformé entièrement en code binaire avant d'être compilé, comme pour un langage compilé.

Le principal avantage est que le code source d'un langage interprété peut fonctionner sur tout ordinateur. Le principal inconvénient est qu'un langage interprété est plus lent qu'un langage compilé, qui peut réutiliser certaines parties de code préalablement compilées.

Grâce à ses bibliothèques performantes et régulièrement mises à jour, Python est particulièrement utilisé dans les environnements scientifiques, pour la manipulation de données et pour l'intelligence artificielle. Par ailleurs Python bénéficie d'une communauté active et de nombreux forums permettant de résoudre tous types de problèmes rencontrés.

La structure du code Python

Chaque programme est composé d'instructions, qui sont comme des phrases avec leur propre syntaxe, leur orthographe et leur grammaire. Ces instructions sont lues de haut en bas et de la gauche vers la droite. Contrairement à la langue française, le nombre de mots est très limité, et la syntaxe est toujours la même. Le meilleur moyen de bien connaître cette syntaxe est de pratiquer le coding en s'aidant des commentaires fournis par l'interpréteur Python pour chaque erreur de code rencontrée. Ainsi chaque erreur est une opportunité pour améliorer sa façon de coder. Il faut par ailleurs garder à l'esprit que l'ordinateur exécute toujours à la lettre ce qu'on lui demande de faire.

Dans un livre, les phrases composent des paragraphes, qui eux-même composent des chapitres. En suivant cette analogie, une ligne de code serait une phrase, un ensemble d'instructions regroupées entre elles (ou bloc) serait un paragraphe, un fichier serait un chapitre et un projet serait un livre.

Le langage Python a la particularité de ne pas séparer les blocs de code par des accolades. Il utilise plutôt un niveau de décalage à droite (appelé indentation) qui représente chaque bloc.

Par exemple, au lieu d'avoir un code structuré ainsi :

{
instruction du bloc 1;
instruction du bloc 1;
{
instruction du bloc 2;
instruction du bloc 2;
}
suite instruction du bloc 1;
}

le même code en Python serait structuré ainsi :

instruction du bloc 1
instruction du bloc 1
    instruction du bloc 2
    instruction du bloc 2
suite instruction du bloc 1

On remarque tout de suite que le code Python est plus court et plus lisible. En revanche, il est impératif de penser à décaler les instructions du bloc 2, sinon elles appartiendraient au bloc 1 et cela changerait forcément le résultat obtenu.

La fin d'une ligne symbolise la fin d'une instruction. Il n'est pas nécessaire en Python d'utiliser des points-virgules pour symboliser la fin d'une instruction.

Par ailleurs, les programmes comprennent des fonctions qu'on appelle autant de fois que nécessaire pour éviter de répéter les mêmes lignes de code.

Il est possible et même très important d'écrire des commentaires dans un programme. Ces portions de code ne sont pas exécutées par l'interpréteur Python et ont pour but de faciliter la compréhension du code pour le développeur, car parfois les codes sont compliqués. Les commentaires d'une ligne commencent toujours par le symbole #, alors que les commentaires de plusieurs lignes sont entourés des symboles """, comme dans l'exemple ci-dessous.

# ceci est un commentaire d'une ligne

"""
ceci est un commentaire
sur plusieurs lignes
"""

Il faut prendre l'habitude d'écrire les commentaires en anglais car Python est écrit en anglais, au début de chaque fichier pour en expliquer le but, ainsi qu'au début de chaque fonction (on appelle ces commentaires des Docstrings dans ce cas).

En résumé

  • les codes Python sont composés de blocs qu'on repère grâce à leur décalage par rapport au bord gauche de la page.
  • les commentaires sont destinés aux développeurs et permettent de mieux comprendre le code
  • les fonctions permettent d'éviter de répéter le code

Bien choisir son outil pour exécuter du Python

On peut construire un programme et l'exécuter en utilisant un site internet disposant d'une console et d'un interpréteur en ligne. Dans ce cas aucune installation de Python n'est requise. On peut également construire un programme Python et l'exécuter à l'aide de l'éditeur intégré de Python IDLE, qui colorie automatiquement le code mais dispose de fonctionnalités très limitées. Il est plus pratique d'utiliser un éditeur de texte tel que Sublim Text ou NotePad++ qui colorient automatiquement le code en fonction du langage choisi et proposent en plus une auto-complétion des instructions. Il faut dans ce cas utiliser le Terminal pour exécuter le code. Il est encore plus efficace d'utiliser un Environnement de Développement Intégré IDE tel que PyCharm, JetBeans ou Visual Studio Code (ce dernier n'étant pas toujours considéré comme un IDE), qui proposent en plus une recherche passive des erreurs de frappe, un debugger (aide pour corriger les bugs) et le paramétrage des outils de versioning (pour conserver l'historique du code). L'exécution du code s'effectue dans ce cas par le biais de l'IDE qui propose le choix des versions de Python présentes sur l'ordinateur. IDLE s'installe automatiquement avec Python, et il faut ouvrir un nouveau module pour créer un fichier

On peut construire un programme Python en utilisant un éditeur de texte tel que Sublim Text, NotePad++ qui colorient automatiquement le code en fonction du langage choisi et proposent une auto-complétion des instructions. On peut ensuite exécuter ce programme à l'aide du Terminal d'un ordinateur (sous Linux et MacOS) ou le Powershell (sous Windows).

En résumé

Il est possible d'exécuter Python en utilisant l'un des outils suivants (du plus pratique au plus performant) :

  • un site internet proposant un interpréteur en ligne
  • l'éditeur intégré de Python IDLE
  • un éditeur de texte et le Terminal pour exécuter le code
  • un environnement de développement intégré IDE

Exécuter son code à l'aide de IDLE

Cette manière de procéder est plus complexe. Elle requiert un peu d'expérience pour télécharger et installer Python sur son ordinateur.

Python est souvent installé nativement sur les ordinateurs, mais sa version permet uniquement de faire fonctionner certaines applications de la machine. Cette version n'est généralement pas accessible à l'utilisateur pour exécuter du code. Il est donc nécessaire d'installer une autre version de Python pour développer ses propres programmes.

Il faut d'abord télécharger et installer une version récente de Python sur sa machine. Pour cela, il faut se rendre sur le site officiel de Python, suivre la documentation, télécharger une version du langage et l'installer. Il existe de nombreux tutoriels expliquant comment installer une version de Python, sachant que pour chaque système d'exploitation l'installation est différente.

A toi de jouer : une fois qu'une version de Python a été installée sur la machine, l'éditeur intégré de Python appelé IDLE est présent et fonctionnel. Il suffit de l'ouvrir pour voir apparaître une console capable d'exécuter séparément des lignes de code. Pour exécuter un programme complet, donc un fichier, il faut ouvrir un nouveau Module, taper une ligne de code comme par exemple :

print("hello world")

puis enregistrer le fichier en lui donnant un nom et un emplacement, par exemple bonjour.py dans le dossier Documents.

Pour exécuter le code il suffit de sélectionner Run Module en haut de la fenêtre IDLE et il devrait apparaître dans la toute première page qui a été ouverte, celle de la console :

hello world

A ce niveau les principaux problèmes que tu peux rencontrer sont les suivants :

  • tu as donné un nom de fichier avec des " et des espaces et le programme s'exécute mal
  • tu ne vois pas le résultat affiché car tu cherches sur ta page Module au lieu de la première page ouverte Console

Exécuter son code à l'aide du Terminal

Comme précédemment, il faut télécharger et installer une version récente de Python sur sa machine. Pour cela, il faut se rendre sur le site officiel de Python, suivre la documentation, télécharger une version du langage et l'installer. Il existe de nombreux tutoriels expliquant comment installer une version de Python, sachant que pour chaque système d'exploitation l'installation est différente.

Il faut ensuite installer un éditeur de texte comme Sublim Text ou NotePad++. Voici le lien vers le site officiel de Sublim Text.

L'exécution d'un programme au Terminal est simple et rapide si les installations de Python et Sublim Text fonctionnent correctement.

A toi de jouer : par exemple, pour afficher "Hello world", il faut se placer dans un dossier comme Documents, creer un fichier appelé bonjour.py dans ce dossier, puis ouvrir ce fichier dans Sublim Text et finalement écrire son premier code :

print("Hello world")

Ensuite, ouvrir le Terminal, et exécuter l'instruction suivante en précisant le chemin d'accès à bonjour.py qui se trouve dans le dossier Documents. Pour exécuter du Python il suffit de commencer par python3. Pour exécuter une instruction au Terminal, il suffit d'appuyer sur la touche Entrée après avoir tapé cette instruction.

python3 ~/Documents/bonjour.py

La plus grosse difficulté et la principale source d'erreurs consiste à se tromper dans l'adresse du fichier ~/Documents/bonjour.py, qui dépend de l'emplacement où se trouve ton fichier.

Si tout fonctionne normalement, tu dois voir s'afficher dans ton Terminal :

Hello world

On constate donc que l'éditeur de texte sert à écrire son code, et le Terminal demande à l'ordinateur d'exécuter le langage Python.

Cette méthode présente deux inconvénients majeurs. Tout d'abord il faut télécharger et installer manuellement toutes les bibliothèques qui n'existent pas nativement dans Python et qu'on souhaite utiliser. C'est le cas par exemple de matplolib ou de biopython. Ensuite, on ne dispose ni d'un correcteur de code intégré, ni d'un debuggeur. Toutefois, il peut être pratique d'exécuter un code directement au Terminal pour vérifier par exemple qu'un problème d'exécution n'est pas lié à l'environnement de développement qu'on utilise. Et puis cela permet de bien comprendre que Python ne fait pas partie de l'environnement de développement ou de l'éditeur de texte, mais qu'il est simplement utilisé par cet environnement de développement.

Exécuter son code à l'aide d'un environnement de développement intégré IDE

La principale difficulté lors de l'utilisation d'un environnement de développement intégré est la suivante : l'exécution d'un programme Python nécessite en général que le développeur paramètre sur son ordinateur le chemin d'accès à l'interpréteur Python, car il y a de fortes chances pour que votre machine dispose de plusieurs versions de Python.

L'installation et la configuration sont dans ce cas plus complexes que pour un simple éditeur de texte et dépendent de chaque IDE. A noter que de nombreux IDE utilisent un espace de travail (workspace) dans lequel sont stockés les fichiers sources et les utilitaires. Attention donc, les fichiers enregistrés ne sont pas toujours là où l'on pense.

Le debugger est un outil particulièrement intéressant, qui propose non seulement des commentaires relatifs aux erreurs de code, des solutions possibles, mais également la possibilité d'exécuter le code de point d'arrêt en point d'arrêt pour connaître l'état des variables du programme à chacune des lignes de son exécution, ces lignes ayant été préalablement définies par le développeur.

Un outil de versioning tel que Git est également très utile puisqu'il permet de conserver un historique des différentes versions de code, de revenir en arrière en choisissant une version plutôt qu'une autre si le code actuel n'est pas satisfaisant. Par ailleurs, il permet de travailler sur des copies de code plutôt que sur des originaux et de regrouper le travail de toute une équipe en évitant les conflits de code.

Découvrir Python avec un jeu de devinette

Voici le code d'un programme Python qui tire un nombre au hasard et demander au joueur de le deviner en 5 tentatives.

from random import randint

nbre_essais_max = 5
tentative = 1
min = 0
max = 50
choisi = randint(min, max)   
proposition = -1                      

print("Devine en " + str(nbre_essais_max) + " le nombre que j'ai choisi, entre " + str(min) + " et " + str(max))

while proposition != choisi and tentative <= nbre_essais_max:
    proposition = int(input("Ta proposition : "))
    if proposition < choisi:
        print("Trop petit")
    elif proposition > choisi:
        print("Trop grand")
    else:
        print("Bravo ! Tu as trouvé " + str(choisi) + " en " + str(tentative) + " essai(s)")
    tentative += 1
        
if tentative > nbre_essais_max and proposition != choisi :
    print("Perdu. J'avais choisi le nombre : " + str(choisi))

Copie-colle le programme ci-dessus dans ce nouveau fichier, donne un nom à ton fichier comme par exemple premierJeu.py, puis exécute le code. Attention, l'extension .PY est importante car elle indique à ton environnement de travail que le programme est écrit en Python.

  • Pour mieux comprendre ce que fait le programme, lit les lignes de codes une à une en essayant de deviner ce qu'elle fait.
  • Si on veut maintenant que le jeu se joue en 10 tentatives, que faut-il changer dans le code ?
  • Si on veut ensuite que le nombre à deviner soit compris entre 1 et 100, que doit-on changer dans le code ?
  • Si on veut finalement truquer le jeu pour que le joueur ne trouve jamais la solution. Comment pourrait-on changer le code ?

Découvrir Python avec une course d'escargots

Voici le code d'un programme Python qui fait courir trois escargots jusqu'à une ligne d'arrivée et affiche vainqueur lorsque le premier escargot a terminé.

from tkinter import *
from time import sleep
from random import randint

def demarrer():
    global x1, x2, x3
    y = 1000
    while x1 < 1040 and x2 < 1040 and x3 < 1040:
        nbre_hasard = randint(1,3)
        if nbre_hasard == 1:
            x1 += 10
            can.coords(esca1, x1, y1)
            y = y1
        elif nbre_hasard == 2:
            x2 += 10
            can.coords(esca2, x2, y2)
            y = y2
        else:
            x3 += 10
            can.coords(esca3, x3, y3)
            y = y3
        sleep(0.05)  
        can.update()
    can.coords(vainqueur, 500, y)


def reinit():
    global x1, x2, x3, y1, y2, y3
    can.coords(esca1, x1, y1)
    can.coords(esca2, x2, y2)
    can.coords(esca3, x3, y3)
    can.coords(vainqueur, 200, 800)

x1, y1  = 130, 75
x2, y2  = 130, 225
x3, y3  = 130, 375

fen = Tk()
fen.title("Course d'escargots")
can = Canvas(fen, width=1100, height=500, bg ='white')
can.pack(side=TOP, padx=5, pady=5)
b2 = Button(fen, text ='Départ de la course', width=15, command=demarrer)
b2.pack(side=RIGHT)

can.create_line(1080, 0, 1080, 500, width=5, fill="red")
can.create_line(250, 0, 250, 500, width=5, fill="green")
can.create_line(0, 150, 1100, 150, width=5, fill="black")
can.create_line(0, 300, 1100, 300, width=5, fill="black")
can.create_line(0, 450, 1100, 450, width=5, fill="black")

photo1 = PhotoImage(file ='~/Documents/ecg1.png')
esca1 = can.create_image(x1, y1, image=photo1)
photo2 = PhotoImage(file ='~/Documents/ecg2.png')
esca2 = can.create_image(x2, y2, image=photo2)
photo3 = PhotoImage(file ='~/Documents/ecg3.png')
esca3 = can.create_image(x3, y3, image=photo3)
photo4 = PhotoImage(file ='~/Documents/vainqueur.png')
vainqueur = can.create_image(200, 800, image=photo4) 

fen.mainloop()
fen.destroy()

Télécharge ensuite les images ecg1.png, ecg2.png, ecg3.png et vainqueur.png, puis place ces images dans le même dossier que ton fichier escargot.py. Exécute ensuite le code pour jouer.

Il est possible que ton programme rencontre des difficultés pour trouver l'adresse des images. Tu peux dans ce cas placer ton fichier escargots.py ainsi que les images dans le dossier Documents et modifier les adresses du type 'ecg1.png' en adresses du type '~/Documents/ecg1.png'
