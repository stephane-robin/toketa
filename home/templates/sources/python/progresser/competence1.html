Le choix du nom des variables

Il est important de construire un code propre, pour comprendre ce que fait le code lorsqu'on l'a abandonné pendant une longue période ou pour faciliter le travail collaboratif. Pour cela, les variables doivent avoir un nom clair ayant une véritable signification.

Par exemple le code suivant n'est pas vraiment lisible :

import math
f = 3
g = 4.5
t = math.sqrt(f ** 2 + g ** 2)

Alors que si les noms des variables avaient une signification claire, le code serait plus lisible. Voici le même code en changeant simplement les noms de variables :

import math
cote1 = 3
cote2 = 4.5
hypotenuse = math.sqrt(f ** 2 + g ** 2)

Les commentaires

On retrouve essentiellement des commentaires au début de chaque fichier (appelé module) pour expliquer l'objectif du programme, puis au début de chaque fonction pour expliquer son rôle et ce dont elle a besoin pour s'exécuter.

Les commentaires sur plusieurs lignes sont entourées de """, alors que les commentaires d'une seule ligne sont précédés de #.

"""This module compares the results obtained between 2 points with 2 kinds of distances, the distance of 
Euclide and the distance of Manhattan."""

import math
M1 = (2, 4)
M2 = (-1, 3)

def calculerEuclide(M, N):
    """finds the length between points M and N which coordinates are given, using square roots.
    @param:
        M (tuple): coordinates of point M 
        N (tuple): coordinates of point N 
    @return:
        (float): distance of Euclide between points M and N
    """
    return math.sqrt((M[0] - N[0]) ** 2 + (M[1] - N[1]) ** 2)

def calculerManhattan(M, N):
    """finds the length between points M and N which coordinates are given, using absolute values.
    @param:
        M (tuple): coordinates of point M 
        N (tuple): coordinates of point N 
    @return:
        (float): distance of Manhattan between points M and N
    """
    return math.sqrt((M[0] - N[0]) ** 2 + (M[1] - N[1]) ** 2)

print("Distance de Euclide : " + calculerEuclide(M, N))
print("Distance de Manhattan : " + calculerManhattan(M, N))

Les commentaires de fonctions appelés docstrings sont essentiels et permettent de comprendre ce que fait une fonction au milieu d'un programme sans avoir à se rendre dans la partie du programme qui définit cette fonction. Les docstrings ne sont pas lues par le programme au moment de son exécution. Ils sont destinés au développeur. On peut donc voir le descriptif d'une fonction qu'on exécute à la ligne 1204 alors qu'elle a été définie à la ligne 30 ou dans autre fichier, comme dans l'exemple ci-dessous.

Il est par ailleurs possible de fournir des informations supplémentaires au programme, comme par exemple le type des variables utilisées dans une fonction ou le type retourné par une fonction. On fournit alors ces indications dans la signature de la fonction, c'est à dire la première ligne de la définition de cette fonction, comme par exemple :

def calculerEuclide(M: float, N: float) -> float:

Python est un langage faiblement typé, c'est à dire qu'il n'impose pas la définition d'un type de variable, et permet le changement de type d'une variable dans un même programme. Toutefois, il n'est pas capable de concaténer une chaine et un nombre ou de les comparer, et dans ce cas il déclenche une erreur.

Tester chaque fonction

Lorsqu'on crée une fonction, il faut immédiatement tester son bon fonctionnement avec des paramètres dont on connaît les résultats qu'ils fournissent. Pour cela, on utilise l'instruction assert, comme dans l'exemple ci-dessous :

def calculerSurface(largeur, longueur):
    if largeur < 0 or longueur < 0:
        return -1
    else:
        return largeur * longueur 

assert calculerSurface(3, 2) == 6, "erreur de calcul de surface"
assert calculerSurface(-3, 2) == -1, "erreur de calcul de surface pour cote negatif"
