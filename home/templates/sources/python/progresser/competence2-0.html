Découvrir les listes

Nous avons vu qu'une chaîne de caractères est un ensemble de caractères qu'on repère par un indice. On peut considérer une liste comme étant un ensemble d'éléments (textes ou nombres) qu'on repère par un indice.

Par exemple, ceci est une liste

maListe = ["lundi", "mardi", "mercredi", "jeudi", "vendredi"]

A RETENIR ...

Dans une liste le 1er élément a pour indice 0 et non pas 1, c'est à dire qu'il se trouve à la position 0.

Par opposition aux types de variables de base comme int, float, str, bool, les listes sont un type de variable structuré complexe noté list en Python.

ancre

Définir une liste

Pour définir une liste vide, on a le choix entre deux possibilités

maListe = []
maListe = list()

Pour définir une liste contenant déjà des éléments, il suffit de les écrire en les séparant à l'aide d'une virgule. Attention, il faut penser à mettre des " pour chaque élément lorsqu'il s'agit d'une chaîne de caractères.

maListe = [3, 5, 2, 7, 8, 1]

On peut définir une liste contenant les mêmes éléments (par exemple 0) en la remplissant automatiquement, comme dans le code ci-dessous :

maListe = 10 * [0]

Il existe une méthode plus avancée pour remplir cette liste, en faisant appel à une boucle for :

liste = [0 for k in range(10)]
,

ce qui nous permet, suivant le même procédé, de cérer une liste composée de nombres pairs par exemple :

liste = [2 * i for i in range(10)]

Pour afficher les listes que nous venons de créer, il suffit d'écrire

print(liste)

A vous de jouer : exécutez les codes ci-dessus pour bien comprendre le mécanisme de création d'une liste. Puis créez une liste composée uniquement de nombres impairs.

Solution :

liste = [2 * i + 1 for i in range(10)]

Les opérations élémentaires sur les listes

Récupérer un élément

Pour récupérer l'élément d'indice i de maListe et le placer dans la variable jour :

jour = maListe[i]

Rajouter un élément en fin de liste

maListe.append("mon nouvel élément")

Attention, l'instruction append() ne permet de rajouter qu'un seul élément à une liste. Par exemple l'instruction suivante engendre une erreur :

mois = ["janvier", "février", "mars"]
mois.append("avril", "mai") # ERREUR

Si on rajoute l'élément ("avril", "mai"), on ne rajoute qu'un seul élément mais il s'agit d'un tuple, ce qui modifie la structure de notre liste, comme ceci :

mois = ["janvier", "février", "mars"]
mois.append(("avril", "mai"))
print(mois) # affiche ["janvier", "février", "mars", ("avril", "mai")]

Il faut donc procéder en plusieurs étapes de la façon suivante :

mois = ["janvier", "février", "mars"]
mois.append("avril")
mois.append("mai")

ou utiliser l'instruction extend() plutôt que append(). Au lieu de prendre comme paramètre un unique élément, extend() prend en paramètre une liste, comme dans l'exemple ci-dessous :

mois = ["janvier", "février", "mars"]
mois.extend(["avril", "mai"])

Changer un élément dans une liste

Pour remplacer l'élément d'indice i de maListe par "élément modifié", il suffit de redéfinir cet élément. Contrairement aux chaînes de caractères, on peut modifier uniquement un élément d'une liste, car chaque élément est considéré comme une variable indépendante :

maListe[i] = "élément modifié"

Parcourir une liste

La boucle for permet de parcourir une liste. On distingue deux structures différentes, l'une basée sur l'indice de chaque élément de la liste, l'autre basée directement sur chaque élément de la liste.

Les deux exemples ci-dessous fournissent le même résultat en utilisant des méthodes différentes. Il faut absolument connaître les deux méthodes, car elles sont plus ou moins pratiques en fonction de chaque situation.

Le nombre d'éléments de maListe est donné par l'instruction len(maListe).

maListe = ["lundi", "mardi", "mercredi", "jeudi", "vendredi"]

# méthode 1 basée sur l'indice i
for i in range(len(maListe)):
    print(maListe[i])

# méthode 2 basée directement sur l'élément lui-même
for elt in maListe:
    print(elt)

A vous de jouer : soit la liste maListe = [2, 1, 5, 4, 3, 6, 7]
En parcourant maListe avec une boucle for, afficher uniquement les éléments pairs de cette liste. On rappelle que l'instruction x % 2 permet de tester si x est divisible par 2.

Solution :

maListe = [2, 1, 5, 4, 3, 6, 7]

for i in range(len(maListe)):
    if maListe[i] % 2 == 0:
        print(maListe[i])

Rappel, le nombre d'éléments de maListe est donné par l'instruction len(maListe).

Dans le code ci-dessous, on cherche à parcourir maListe et si on trouve l'élément "bonjour", alors on le modifie dans la liste en "hello".

for i in range(len(maListe)):
    if maListe[i] == "bonjour":
        maListe[i] = "hello"

A vous de jouer : considérons la liste suivante

1. Ajouter à chacun des éléments de la maListe la lettre s
2. Afficher chaque élément de maListe en y rajoutant la lettre s, et sans modifier maListe

Notons qu'on utilise systématiquement le parcours de liste avec son indice pour modifier une liste. Cette méthode permet de récupérer, d'utiliser, d'afficher et de modifier les éléments d'une liste, alors que le parcours de liste élément par élément ne modifie pas la liste.

A vous de jouer : pour comprendre le principe énoncé ci-dessus, exécutez le code suivant.

jours = ["lundi", "mardi", "mercredi"]

for elt in jours:
    if elt == "mardi":
        elt = "Tuesday"

print(jours) # affiche ["lundi", "mardi", "mercredi"]

for i in range(len(jours)):
    if jours[i] == "mardi":
        jours[i] = "Tuesday"

print(jours) # affiche ["lundi", "Tuesday", "mercredi"]

Le parcours par élément récupère chaque élément et place sa valeur dans une variable appelée elt. Il affecte alors la valeur "Tuesday" à elt lorsque celui-ci vaut "mardi", puis au tour suivant elt prend la valeur "mercredi". elt est une variable utilisable dans le programme mais n'affecte pas le contenu de jours qui n'a jamais été modifé. En revanche l'affectation jours[i] = "Tuesday" dans le parcours par indice modifie le contenu de jours.

Affectation par valeurs, affectation par référence

Il s'agit d'une subtilité dans l'utilisation des listes, source de beaucoup d'erreurs de logique, difficiles à corriger.

Pour les types de variables construits (objets, listes, ...), on peut affecter une variable par référence, ou par valeur.

Par référence signifie qu'on utilise l'adresse mémoire de la variable pour lui affecter une valeur. Par exemple :

liste1 = liste2

est une affectation par référence car les données de la liste1 se trouvent au même emplacement mémoire que les données de la liste2.

Par valeur signifie qu'on utilise directement une donnée pour affecter une valeur à la variable. Par exemple :

liste1 = [4, 5, 2, 1]

est une affectation par valeur car les données de la liste1 sont fournies directement à la variable et ne dépendent pas de l'adresse mémoire d'une autre variable.

Le piège est le suivant : lorsqu'on affecte une variable liste1 par référence à une variable liste2, toute modification de liste1 entraîne le changement de liste2 et vice versa.

A vous de jouer : pour comprendre ce principe, exécutez le code suivant :

# AFFECTATION PAR REFERENCE
liste1 = ['x', 'y', 'z']
liste2 = liste1 
liste2[0] = 'a'
print(liste1) # affiche ['a', 'y', 'z']
print(liste2) # affiche ['a', 'y', 'z']

# AFFECTATION PAR VALEUR
liste3 = ['x', 'y', 'z']
liste4 = liste3.copy() 
liste4[0] = 'a'
print(liste3) # affiche ['x', 'y', 'z']
print(liste4) # affiche ['a', 'y', 'z']

On remarque que dans le deuxième cas, on a juste copié les valeurs une à une, au lieu de pointer vers l'adresse de liste3.

Il faut comprendre que cette subtilité n'existe que pour les types de variables construits. Ainsi, dans le code suivant, si on modifie b, cela ne change pas a :

a = 12
b = a
b = 5
print(a) # affiche 12
print(b) # affiche 5

Egalité par référence ou par valeur

On peut être amené à se poser la question suivante : est-ce que 2 listes qu'on souhaite comparer sont égales par référence ou par valeur ?

Si elles sont égales par référence, cela signifie que leurs adresses mémoires sont identiques. On utilise dans ce cas l'opérateur is.

Si elles sont égales par valeur, cela signifie que tous les éléments des listes sont les mêmes individuellement, mais que leurs emplacements mémoire diffèrent. On utilise dans ce cas l'opérateur ==

Ainsi, si l'on reprend l'exemple précédent, on obtient :

print(liste1 == liste2) # affiche True
print(liste1 is liste2) # affiche True
print(liste3 == liste4) # affiche True
print(liste3 is liste4) # affiche False

La notion de tuple

En Python, il existe un type particulier appelé tuple, qui pourrait s'apparenter à une liste car il s'agit d'un regroupement de variables. Toutefois, les tuples sont immuables, c'est à dire qu'on ne peut pas les agrandir, les diminuer ou les modifier.

Alors pourquoi utiliser des tuples si on ne peut pas les modifier ? Les tuples occupent moins d'espace mémoire que les listes car au moment de leur création, il n'est pas nécessaire d'allouer de l'espace de stockage supplémentaire pour gérer leur agrandissement. Les tuples sont essentillement utilisés pour afficher ou conserver un résultat sous la forme de plusieurs valeurs liées. Par exemple :

groupes = ("1B1", "1B2")

est un tuple. On remarque l'utilisation de () au lieu de [] pour une liste. On peut afficher "1B1" de la même façon que pour une liste avec l'instruction :

print(groupes[0])

En revanche, on ne peut pas rajouter ou enlever un élément au tuple groupes. On ne peut pas modifier "1B1" en "1B3" par exemple. En effet, l'instruction :

groupes[0] = "1B3" # ERREUR

génère une erreur de type TypeError: 'tuple' object does not support item assignment.

Voici un autre exemple d'utilisation de tuples avec une fonction qui crée un point dont les coordonnées sont aléatoires :

from random import *

def creer_point():
    return (randint(0, 100), randint(0, 100))

pointA = creer_point()
print(pointA) # affiche les coordonnees du point A

En résumé

  • les tuples offrent l'avantage d'occuper moins d'espace mémoire que les listes, vu qu'au moment de leur création, il n'est pas nécessaire d'allouer de l'espace de stockage supplémentaire pour gérer leur agrandissement.

Exercice 2 : on considère le dictionnaire suivant :

rangement_alphabet = {
    'a': [],
    'b': [],
    'c': []
}

et la liste suivante :

mots = ["avenir", "choix", "bienvenue", "atelier", "aventure"]

Créer une fonction placer_mots() qui prend en paramètre une liste comme mots et range chacun des mots de cette liste dans le dictionnaire rangement_alphabet en les regroupant suivant leur première lettre. Si l'on exécute :

placer_mots(mots)

on obtient donc :

{'a': ['avenir', 'atelier', 'aventure'], 'b': ['bienvenue'], 'c': ['choix']}

Solution

def placer_mots(liste):
    global rangement_alphabet
    for elt in liste:
        if elt[0] == 'a':
            rangement_alphabet['a'].append(elt)
        elif elt[0] == 'b':
            rangement_alphabet['b'].append(elt)
        elif elt[0] == 'c':
            rangement_alphabet['c'].append(elt)
